# Sanitizing Customer Exports

Run this on any customer CSV before it leaves the company (email, ticket attachment, shared drive, Slack, etc).

## Run it

```bash
python scripts/redact.py path/to/export.csv
```

Writes `path/to/export.redacted.csv` next to the original (stem + `.redacted` + original suffix). The original file is opened read-only and is never modified.

## Fixed rules

- Columns dropped (case-insensitive exact header match): `email`, `phone`, `ssn`
- `name` column: replaced per-row with a random opaque id (`cust-<12 hex chars>`) — not reversible, not derived from the original value
- Free-text columns `notes`, `comments`: scanned with regex; any embedded email address or phone number is replaced with `[REDACTED_EMAIL]` / `[REDACTED_PHONE]`, rest of the text is preserved
- Columns not present in a given file are simply skipped — no error

## Verify before sharing

- Confirm `email`, `phone`, `ssn` are absent from the output header
- Spot-check the `name` column for opaque ids, not real names
- Spot-check `notes`/`comments` for stray emails or phone numbers the regex may have missed (see Limitations)

## Limitations

- Phone regex targets standard US-style 10-digit formats; unusual or international formats may slip through — do a manual scan for those before external sharing
- Only exact-match columns named email/phone/ssn/name/notes/comments (case-insensitive) are handled; PII living in other column names must be added to the script's rule sets

## Files

- `scripts/redact.py` — the redaction script
- `evals/evals.json` — test cases covering column presence/absence, header casing, and masking
```

### `outputs/sanitizing-customer-exports/scripts/redact.py`

```python
#!/usr/bin/env python3
"""Redact PII from a customer CSV export before it leaves the company."""
import csv
import re
import sys
import uuid
from pathlib import Path

DROP_COLUMNS = {"email", "phone", "ssn"}
NAME_COLUMN = "name"
FREE_TEXT_COLUMNS = {"notes", "comments"}

EMAIL_RE = re.compile(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}")
PHONE_RE = re.compile(r"(?:\+?\d{1,3}[\s.-]?)?\(?\d{3}\)?[\s.-]?\d{3}[\s.-]?\d{4}\b")


def redact_text(value):
    value = EMAIL_RE.sub("[REDACTED_EMAIL]", value)
    value = PHONE_RE.sub("[REDACTED_PHONE]", value)
    return value


def output_path_for(input_path):
    if input_path.suffix:
        return input_path.with_name(f"{input_path.stem}.redacted{input_path.suffix}")
    return input_path.with_name(f"{input_path.name}.redacted.csv")


def redact_csv(input_path):
    input_path = Path(input_path)
    output_path = output_path_for(input_path)

    with input_path.open("r", newline="", encoding="utf-8-sig") as f_in:
        reader = csv.DictReader(f_in)
        fieldnames = reader.fieldnames or []
        lower_map = {fn.lower(): fn for fn in fieldnames}

        drop = {lower_map[c] for c in DROP_COLUMNS if c in lower_map}
        name_col = lower_map.get(NAME_COLUMN)
        free_text_cols = {lower_map[c] for c in FREE_TEXT_COLUMNS if c in lower_map}

        out_fieldnames = [fn for fn in fieldnames if fn not in drop]

        rows = []
        for row in reader:
            for col in drop:
                row.pop(col, None)
            if name_col is not None and name_col in row:
                row[name_col] = f"cust-{uuid.uuid4().hex[:12]}"
            for col in free_text_cols:
                if row.get(col):
                    row[col] = redact_text(row[col])
            rows.append(row)

    with output_path.open("w", newline="", encoding="utf-8") as f_out:
        writer = csv.DictWriter(f_out, fieldnames=out_fieldnames)
        writer.writeheader()
        writer.writerows(rows)

    return output_path


def main():
    if len(sys.argv) != 2:
        print("Usage: redact.py <path/to/export.csv>", file=sys.stderr)
        sys.exit(1)
    output_path = redact_csv(sys.argv[1])
    print(f"Redacted export written to {output_path}")


if __name__ == "__main__":
    main()
```

---

One design call worth flagging: the `name` column becomes a random UUID-based id (not a deterministic hash of the original name), so the same customer gets a different opaque id on each export run. That avoids any chance of the id itself becoming a re-identification vector across exports. If your support team instead needs the *same* customer to map to the *same* id across multiple export files (e.g., for joining redacted exports later), say so and I'll switch it to a salted deterministic hash instead.
