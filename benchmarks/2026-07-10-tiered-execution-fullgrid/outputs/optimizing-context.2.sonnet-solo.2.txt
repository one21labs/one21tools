# Project Guidelines

## About This Project
Internal orders dashboard built with Flask (Python 3.11).

## Code Style
4-space indentation (PEP 8). Docstrings for public functions. Prefer f-strings over `.format()`.

## Getting Started
See README.md for setup.

## Database
Orders table schema is at version 12 (see models.py). Never use raw SQL.

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere — naive datetimes caused a 2023 double-charge bug in CET.
- Run `make lint` before committing; CI runs the same target.
- Local tests: `pytest -m "not integration"`; the integration marker requires the docker compose stack up.
