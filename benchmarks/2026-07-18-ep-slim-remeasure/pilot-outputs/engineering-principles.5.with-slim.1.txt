## tasks.md

### Blocking decisions (resolve before build starts — own these today, not "ship next week")
1. **Authorization model** — self-service only (user exports their own data), admin-triggered, or both? This determines every authz check in the endpoint and is the single highest-risk gap in a "export all of a user's data" feature (IDOR / broken access control).
2. **Data scope** — enumerate the actual sources: profile, activity/audit logs, uploaded files, billing, third-party-linked data, etc. "All of a user's data" isn't a spec until someone lists the tables/services.
3. **Format requirement** — "whatever is easiest" isn't a contract. Check whether this needs to satisfy GDPR Art. 20 / CCPA portability (machine-readable, structured) rather than just picking JSON vs CSV for convenience.
4. **Delivery mechanism for "fast"** — synchronous response vs. async job + expiring download link. "Fast" for a small profile blob and "fast" for full account history are different architectures; pick one before estimating.
5. **Abuse/rate limiting** — bulk personal-data export endpoints are a common scraping/DoS target; needs a limit per user/IP.
6. **Export artifact security** — encryption at rest/in transit, retention window, and audit log of who exported what and when.

### Implementation tasks (once the above are answered)
- Define API contract: route, auth requirements, request/response schema
- Implement authz checks per the chosen model (+ negative tests for cross-user access)
- Implement data aggregation across the confirmed data sources
- Implement serialization to the confirmed format
- If async: implement job queue, storage, and expiring download link + notification
- Add rate limiting
- Add audit logging of export events
- Unit + integration tests, including authz edge cases
- Load/perf test against the actual "fast" target once defined
- Deploy behind a flag, monitor first-week usage, then full rollout

Items 1 and 2 are the ones most likely to blow the next-week timeline if left undecided — worth getting explicit answers on those two before any code gets written.
