# checkout-service CLAUDE.md

## Stack & Setup

**Stack:** Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard `npm` workflow.

**Onboarding** (full details in README.md): `clone` → `npm install` → `cp .env.example .env` → `npm run dev`.

## Critical Constraints

### Money: Integer Cents Only

All monetary amounts must be integer cents. **Never use floats for money.**

**Why:** A float-rounding incident in 2024 corrupted invoices. This is a hard constraint.

**How to apply:**
- Validate user input and API payloads as integers.
- All database values are integers; keep them that way through calculations.
- For display, divide by 100 only at the UI layer — do not store the result.

### Migrations: Use `./scripts/migrate.sh`

**Never run `prisma migrate` directly.** Always use `./scripts/migrate.sh`.

**Why:** CI relies on a shadow database that the script manages. Direct `prisma migrate` breaks the shadow database.

### Testing

- **E2E tests** (`npm test`): Runs Playwright suite. **Requires a running dev server.** Start `npm run dev` in another terminal first, or tests fail.
- **Unit tests** (`npm run test:unit`): Runs in isolation.

## Authentication

API routes authenticate with a custom `X-Session-Token` header (not standard Bearer tokens). Full scheme documented in `docs/api.md`.

## Style & Structure

See `docs/style-guide.md` (45 pages). Two exceptions to typical TypeScript:

- **No default exports** — all exports are named.
- **Zod schemas colocated with route handlers** — define schemas next to the routes that use them.

## Deployment

Identical across all six company repos:

1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency > 500ms
7. `./deploy prod`
8. Announce in #releases

## History

Started as Rails prototype (2021), rewritten to Next.js in 2023 by platform team. Now owned by payments squad.
