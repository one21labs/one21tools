```markdown
# Atlas Monorepo

Atlas is three independently deployable services in one repo:

- `api/` ŌĆö Go, gRPC backend. See `api/CLAUDE.md`.
- `web/` ŌĆö React/TypeScript frontend. See `web/CLAUDE.md`.
- `pipeline/` ŌĆö Python/Airflow data pipeline. See `pipeline/CLAUDE.md`.

Read the service's own CLAUDE.md before working in that directory ŌĆö it has the commands and quirks that matter there.

## Org-wide rules (apply to all services)

- **Commits**: Conventional Commits format (`feat:`, `fix:`, `chore:`, etc.) on every commit, no exceptions.
- **Before any PR**: `make check` must pass at the repo root. Never open or update a PR without running it first.
- **Branching**: Trunk-based development. No long-lived branches ŌĆö small PRs merged directly to main; feature-flag anything that has to land incomplete.

## Incident response

For a live incident, use the incident-response skill (or `/incident`) ŌĆö it's the same 20-step runbook every team at the company follows. Don't improvise a different process.

## Analytics / warehouse SQL

A read-only SQL connector to the analytics warehouse is available for product-metrics questions. Only run `SELECT` queries ŌĆö never attempt writes or schema changes through it.

## Working across services

- Don't hand-edit anything under `api/gen/` ŌĆö it's regenerated by `make proto` and edits are silently lost.
- Default to `npm run test:unit` in `web/`; only run `npm test` (Playwright, real browsers) when explicitly asked for end-to-end coverage.
- Any command that imports or tests Airflow DAGs in `pipeline/` requires `AIRFLOW_HOME=pipeline/.airflow` to be set first.
```
