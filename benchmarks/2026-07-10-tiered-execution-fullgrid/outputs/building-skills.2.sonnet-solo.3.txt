# JSON Config Check

## Purpose

Validate a JSON configuration file for syntax errors, and if a corresponding JSON Schema is available, validate the file's structure and types against that schema.

## Instructions

1. Identify the target JSON file from the user's request (ask for the path if ambiguous).
2. Check for syntax errors first:
   - Run `python3 -c "import json; json.load(open('<path>'))"` (or use the `json` module directly if scripting).
   - If parsing fails, report the exact error message and line/column number to the user and stop — do not attempt schema validation on invalid JSON.
3. Look for a schema to validate against, in this order:
   - A file referenced by a `$schema` key inside the JSON file itself.
   - A file in the same directory named `<config-name>.schema.json` or `schema.json`.
   - If none is found, skip schema validation and report the file as syntactically valid JSON with no schema found.
4. If a schema is found, validate with the `jsonschema` package:
   ```python
   import json, jsonschema
   config = json.load(open("<config-path>"))
   schema = json.load(open("<schema-path>"))
   jsonschema.validate(instance=config, schema=schema)
   ```
   Report any `ValidationError`, including the failing path (e.g., `config['database']['port']`) and the reason.
5. Summarize results for the user: whether the file is valid JSON, whether a schema was found, and whether schema validation passed or failed (with specifics).

## Notes

- `jsonschema` is a third-party package (`pip install jsonschema`). If it isn't available, fall back to reporting syntax-only validation rather than failing silently.
- This skill only reports issues; it does not modify the config file.
