```markdown
# Project Guidelines

## Overview
Flask web application serving an internal orders dashboard. Python 3.11.

## Code Style
- 4 spaces for indentation (PEP 8)
- Prefer f-strings over `.format()`
- Write docstrings for public functions
- Keep functions short and focused
- Meaningful variable names

## Database
- Orders table schema at version 12 (models.py)
- Never use raw SQL

## Critical Conventions
- **Always use `datetime.now(tz=timezone.utc)`** — naive datetimes caused a 2023 production bug that double-charged customers in CET timezones
- Run `make lint` before committing (CI runs the same target)
- Local tests: `pytest -m "not integration"`
- Integration tests need the docker compose stack running

## Setup
See README.md for installation and running the server.
```
