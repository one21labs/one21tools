```markdown
# checkout-service

Next.js 14 (App Router) + TypeScript + Tailwind + Prisma/Postgres checkout service, owned by the payments squad.

## Key Decisions

- All monetary amounts are integer cents. Never use floats for money — a 2024 float-rounding bug corrupted invoices.
- No default exports.
- Zod schemas are colocated next to the route handlers they validate, not in a shared schemas directory.
- API auth uses a custom `X-Session-Token` header, not Bearer tokens — full scheme in docs/api.md.

## Common Mistakes

- Never run `prisma migrate` directly — it breaks the shadow database CI depends on. Always use `./scripts/migrate.sh` for migrations.
- `npm test` runs the Playwright E2E suite and requires a running dev server. For unit tests, use `npm run test:unit` instead.

## Commands

- Unit tests: `npm run test:unit`
- E2E tests (needs dev server running): `npm test`
- Migrations: `./scripts/migrate.sh`

## Deploy

1. Merge to main, wait for CI green
2. `npm run build:prod`
3. `./deploy staging`, smoke-test `/healthz`
4. Page on-call if p95 latency exceeds 500ms
5. `./deploy prod`
6. Announce in #releases

## Where to Look

- Full style guide: docs/style-guide.md (only deviations from standard TypeScript style are the no-default-exports and colocated-Zod-schema rules above)
- Auth scheme details: docs/api.md
- Onboarding/setup: README.md
```
