**`CLAUDE.md`**
```markdown
# CLAUDE.md

## Code Style
- Prefer `const` over `let`.
- Use meaningful, descriptive names.
- Components: PascalCase. Hooks: prefixed with `use`.
- Keep functions small.
- Avoid deeply nested ternaries.

## API Authentication — /api/* routes
Requests to `/api/*` require a custom `X-Client-Sig` header. **This is not Bearer auth — do not write `Authorization: Bearer ...` for these routes, it will fail.**
- `X-Client-Sig` = HMAC-SHA256 of the request body, signed with the key from `scripts/sign.mjs`.
- Full details: `docs/auth.md`.

## Tests
- `pnpm test:unit` — fast, use this while iterating.
- `pnpm test` — full e2e suite. Launches real Chromium instances and requires the dev server running. Not for quick iteration.

## Setup
1. Clone the repo
2. `pnpm install`
3. Copy `.env.example` to `.env`
4. `pnpm dev`
5. Open `localhost:3000`

(Mirrors README.md — update both if this changes.)

## Deploying
Deploys follow a strict, ordered runbook — see [runbook.md](runbook.md). Do not skip steps, especially the bundle-size gate (step 3) and the canary error-rate/rollback threshold (step 11).
```

**`runbook.md`**
```markdown
# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card `4242`.
9. Post staging sign-off in `#launch`.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes — rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in `#releases`.
```
