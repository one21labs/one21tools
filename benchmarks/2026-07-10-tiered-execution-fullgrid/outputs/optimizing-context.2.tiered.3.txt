```markdown
# Project Guidelines

## About This Project
Python web application built with Flask. Serves an internal orders dashboard.

## Code Style
- Python 3.11, 4 spaces per PEP 8
- Meaningful variable names
- Docstrings for public functions
- Keep functions short and focused
- Prefer f-strings over `.format()`

## Database
Orders table schema is at version 12 (models.py). Never use raw SQL.

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere — a 2023 bug from naive datetimes double-charged customers in CET
- Run `make lint` before committing; CI runs the exact same target
- Local tests: `pytest -m "not integration"` — integration tests need the docker compose stack up
```
