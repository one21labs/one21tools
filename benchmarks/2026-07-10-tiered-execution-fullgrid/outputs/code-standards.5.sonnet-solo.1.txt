```python
# webhook.py

def handle_invoice_event(ledger, payload):
    if payload["event_type"] == "invoice.paid":
        invoice = payload["invoice"]
        ledger[invoice["id"]] = invoice["amount_cents"]
```

One note, not a validation concern but a distinct one: schema validation (are the fields present/well-formed) and authentication (did this request actually come from the billing provider, e.g. HMAC signature check on the raw request body) are different things. Skipping the former based on "the provider validates the event" is reasonable if this function only ever receives payloads that already passed signature verification upstream — worth confirming that check exists somewhere in the request path before this handler is called, since without it anyone who can POST to the endpoint could credit arbitrary amounts to arbitrary invoice ids.
