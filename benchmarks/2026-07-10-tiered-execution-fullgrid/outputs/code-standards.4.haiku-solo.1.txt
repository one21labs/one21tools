Here's the fixed module with minimal exception handling to skip malformed lines:

```python
def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, IndexError):
            # Skip malformed lines and continue processing
            pass
    return prices
```

This catches malformed lines (wrong column count, non-numeric price) and continues the batch instead of crashing. It's the absolute minimum change needed to unblock production—no refactoring, no logging, no configuration.
