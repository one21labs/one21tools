```markdown
# CLAUDE.md — checkout-service

Payment checkout service owned by the payments squad. Next.js 14 (App Router),
TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

## Non-negotiable rules

- **Money is always integer cents.** Never use floats for monetary amounts. A
  2024 float-rounding incident corrupted invoices; floats for money are banned.
- **Run migrations only through `./scripts/migrate.sh`.** Never call
  `prisma migrate` directly — it breaks the shadow database CI depends on.
- **No default exports.** Use named exports everywhere.
- **Colocate Zod schemas next to their route handlers** (not in a shared
  schemas directory).

## Commands

- `npm run dev` — start the dev server.
- `npm run test:unit` — unit tests.
- `npm test` — Playwright E2E suite. **Requires a running dev server** and will
  fail without one. Start `npm run dev` first (or expect failures).
- `./scripts/migrate.sh` — the only supported way to run migrations.
- `npm run build:prod` — production build (used in the deploy flow below).

## Auth

API routes authenticate with a **custom `X-Session-Token` header** — not
standard Bearer tokens. Don't assume OAuth/Bearer conventions. Full scheme is
in `docs/api.md`.

## Style

Full style guide (45 pages) lives at `docs/style-guide.md`. It's standard
TypeScript style with only two deviations, both already listed under
Non-negotiable rules above: no default exports, and Zod schemas colocated with
route handlers.

## Deploy procedure

Identical across all six company repos:

1. Merge to `main`.
2. Wait for CI to go green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. If p95 latency exceeds 500ms, page on-call.
7. `./deploy prod`.
8. Announce in `#releases`.

## Reference

- Setup / onboarding: see `README.md` (clone, `npm install`, copy
  `.env.example` to `.env`, `npm run dev`).
- API & auth details: `docs/api.md`.
- Style guide: `docs/style-guide.md`.
```
