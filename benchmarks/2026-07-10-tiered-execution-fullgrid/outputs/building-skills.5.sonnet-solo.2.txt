```markdown
# Generating Release Notes

## Process

1. Find the last release tag:
   `git describe --tags --abbrev=0`
2. Collect commits since that tag:
   `git log <tag>..HEAD --oneline`
3. Group the commits by their conventional-commit type prefix (see mapping below). Each non-empty group becomes a section.
4. Write the result as markdown to `RELEASE_NOTES.md` at the repo root.

## Commit Type Mapping

| Prefix | Section | Notes |
|---|---|---|
| `feat` | Features | |
| `fix` | Bug Fixes | |
| `perf` | Performance | |
| `docs` | *(omit)* | |
| `chore` | *(omit)* | |
| `ci` | *(omit)* | |
| `build` | *(omit)* | |
| `refactor` | *(omit)* | unless user-facing |
| `revert` | Reverts | also remove the reverted entry if it's in the same release |
| any type with `!` (e.g. `feat!`) | Breaking Changes | always the first section |
| `fix(security)` or any security-scoped commit | Security | always listed before Bug Fixes |

## Section Order

1. Breaking Changes
2. Security
3. Features
4. Performance
5. Bug Fixes
6. Reverts

Omit any section with no entries.

## Entry Format

- One line per entry, imperative mood, no trailing period
- Every entry links its PR number as `[#123]`

## Background

For the definitions (release notes, semver, git tags) and the history of how this process evolved, see `BACKGROUND.md`.
```

```markdown
# Background

## Definitions

- **Release notes**: a document distributed with a software release describing the changes, improvements, and fixes it contains, so users and stakeholders understand what changed.
- **Semantic versioning (semver)**: version numbers as MAJOR.MINOR.PATCH — MAJOR for breaking changes, MINOR for new backwards-compatible functionality, PATCH for backwards-compatible bug fixes.
- **Git tag**: a marker on a specific point in a git repo's history, typically used to mark a release.

## History of this process

- Originally written by hand in Confluence — took hours every sprint.
- Q3 2025: replaced with a shell script dumping raw `git log` output — output was unreadable.
- January 2026: Priya rewrote the process around conventional commits.
- March 2026: the old script was retired after the mobile team migrated.
- This skill replaces that process.
```
