```markdown
# JSON Config Check

## Steps

1. Parse the target file with `json.load`. On `json.JSONDecodeError`, report the line, column, and message — stop here; do not attempt schema validation on invalid JSON.
2. Look for a schema: a `$schema` key inside the config pointing to a file, or a sibling file named `<config-name>.schema.json`. If neither exists, report the config is syntactically valid and stop.
3. Validate with `jsonschema`, collecting all `ValidationError`s (don't stop at the first).
4. Report each error as `<json-path>: <message>`.

## Code

\`\`\`python
import json
import jsonschema

def check(config_path, schema_path=None):
    with open(config_path) as f:
        try:
            config = json.load(f)
        except json.JSONDecodeError as e:
            return [f"Syntax error at line {e.lineno}, column {e.colno}: {e.msg}"]

    if not schema_path:
        return []  # valid JSON, no schema to check against

    with open(schema_path) as f:
        schema = json.load(f)

    validator = jsonschema.Draft202012Validator(schema)
    errors = sorted(validator.iter_errors(config), key=lambda e: list(e.path))
    return [f"{'.'.join(map(str, e.path)) or '<root>'}: {e.message}" for e in errors]
\`\`\`

## Output

Report a syntax error and stop if parsing fails. Otherwise list each schema violation as `path: message`, or state that the config is valid.
```
