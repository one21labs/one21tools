On the design question itself: for most web apps, I'd recommend a **sliding idle timeout of 15–30 minutes** (reset on each authenticated request) **plus a hard absolute session cap of 8–24 hours** regardless of activity. The idle timeout limits exposure from an unattended logged-in browser; the absolute cap bounds how long a stolen/leaked token stays valid even if it's kept "alive." Store both an `issuedAt` and `lastSeenAt` on the session/token, and have the middleware reject (401 → redirect to login) if `now - lastSeenAt > idleTimeout` or `now - issuedAt > absoluteTimeout`, updating `lastSeenAt` on each valid request.

Tighten the idle window (5–15 min) if this handles sensitive data (financial, health, admin panels); a consumer app with low-risk data can go longer (30–60 min idle, multi-day absolute).
