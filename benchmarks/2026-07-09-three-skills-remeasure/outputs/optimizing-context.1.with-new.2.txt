```markdown
# CLAUDE.md

Checkout service (payments squad) — Next.js 14 App Router, TypeScript, Tailwind, Prisma/Postgres.

## Key Decisions

- All monetary amounts are integer cents. **Never use floats for money** — a 2024 float-rounding bug corrupted invoices.
- Auth uses a custom `X-Session-Token` header scheme, not standard Bearer tokens. Full scheme is in docs/api.md — read it before touching auth code.
- Style deviations from typical TypeScript (rest of style guide is standard, see docs/style-guide.md):
  - No default exports.
  - Zod schemas are colocated next to their route handlers.

## Common Mistakes

- `npm test` runs the Playwright E2E suite and requires a running dev server — it will fail otherwise. Use `npm run test:unit` for unit tests instead.
- Never run `prisma migrate` directly — it breaks the shadow database CI relies on. Always go through `./scripts/migrate.sh`.

## Commands

- `npm run test:unit` — unit tests
- `npm test` — Playwright E2E (start `npm run dev` first)
- `./scripts/migrate.sh` — migrations

## Where to Look

- docs/api.md — auth scheme details
- docs/style-guide.md — full style guide
- README.md — setup/onboarding

## Deploys

merge to main → CI green → `npm run build:prod` → `./deploy staging` → smoke-test `/healthz` → page on-call if p95 > 500ms → `./deploy prod` → announce in #releases.

This procedure is identical across all six company repos — since it's duplicated by hand right now, it'd be worth extracting into one shared runbook/skill rather than maintaining six copies that can drift.
```
