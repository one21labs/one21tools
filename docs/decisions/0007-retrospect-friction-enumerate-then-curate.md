# 0007 — retrospect friction handoff: independent git cross-check

- Date: 2026-06-29
- Owner: PM
- Panel: 3 fresh per-call lenses — A (epistemics/structural-bias), B (cost/feasibility/muda), C (portability/poka-yoke/placement); all CONVERGED against a transcript default. `verifier` + `red-team` ran: verifier PASS (2 precision fixes folded); red-team landed a correct HIGH break — the original enumerate-then-curate fix was COSMETIC for the demonstrated mode (a perception failure UPSTREAM of enumerate-vs-curate) — revised in place per that break. Framework repo, inherits 0001/0002; lenses per 0004. Dogfooded `/decide` BY HAND (the plugin's `/decide` is not installed this session).
- Context: a prior `/retrospect` DROPPED its highest-cost finding (an undocumented squash-merge trap) from the friction list handed to the `retrospect` agent. Root cause (red-team, accepted): the orchestrator never PERCEIVED it as friction — upstream of any curate step. The agent cannot see the chat and the orchestrator authors AND curates the list in one ephemeral turn (`retrospect.md:12-13`, `retrospect/SKILL.md:25-27`; agent tools `retrospect.md:5` = Read/Grep/Glob/Bash, no chat access). Proposed fix (user ask): make raw/abridged transcript handoff the DEFAULT. Open question: should the contract default to a transcript instead of the curated list?

## Decision
1. **REJECT transcript-as-default for the shipped contract** — partly rejecting the framing. The only orchestrator->subagent channel is the spawn prompt; anything passed is orchestrator-SELECTED, so "pass an abridged transcript" is operationally identical to "curate less aggressively" (B) — it renames curation, never severs curator from worker. A *raw* transcript default also harms every consumer: SECRETS (keys/tokens/customer data exfiltrated into a subagent every PR), CONTEXT/COST (window blowup against the tier just cut to sonnet, ADR 0006), SIGNAL DILUTION (C). "Abridged caught it this session" is ONE draw from a graded-own-homework channel, not reproducible (A). Survives the gate intact.
2. **PRIMARY fix (the real poka-yoke): make the retrospect AGENT a second, independent reader.** Add a Method clause: the agent CROSS-CHECKS the orchestrator-supplied friction list against the rework / fix-of-a-fix / revert / force-push signals it independently finds in git, and FLAGS any git-visible friction ABSENT from the supplied list. Independent DATA SOURCE (git, not the orchestrator's perception) = a structurally-separate witness, at zero new channel/secrets/cost. Home: `retrospect.md` Method (it is the only actor that sees git). EXACT wording in [verified] below.
3. **SECONDARY (honest bookkeeping, NOT the cure): enumerate-before-dedupe in the orchestrator's step 4** — list every DISTINCT perceived friction before deduping (the agent dedupes at step 5). This only re-surfaces friction the orchestrator ALREADY perceived; it does nothing for the demonstrated perception miss — that is what (2) is for. Home: `SKILL.md` step 4+5 (the handoff constraint stays here; the detector lives in the agent — two different facts, not a one-home violation).
4. No version bump (CLAUDE.md exempts meta/tooling).

## Justification
The break is correct: the trap was never perceived, so a fix that only re-orders perceive-then-cut is theater — worse, a "fixed" label suppresses user vigilance = detect-not-prevent, which CLAUDE.md forbids. The cure must add a witness with an INDEPENDENT data source. The agent already runs git signal and calls rework "the loudest waste signal" (`retrospect.md:15-20`), so the cross-check is a near-free extension that catches the loudest omission class structurally, and makes the gated escalation trigger OPERABLE (it can now detect a dropped git-visible friction). HONEST SCOPE: git cross-check catches only git-VISIBLE friction; the squash-merge trap was partly a derivation-effort friction that may leave no fix-commit fingerprint — so this closes the loudest class, not all perception failures; the residual stays under the gated extractor trigger. Transcript-as-default is rejected: higher cost, same-author selection persists, no structural gain (B/C). cost x risk x value favors the agent cross-check now + the extractor as a triggered revisit.

## Rejected alternatives
- **Enumerate-then-curate as the PRIMARY fix (original 0007 Decision)** — COSMETIC for the demonstrated mode: the orchestrator authors+curates in one turn with no artifact and no second reader, so it only re-surfaces already-perceived friction; re-shipping the defect under a "fixed" label is detect-not-prevent. Demoted to honest bookkeeping.
- **Raw / abridged transcript to retrospect (A's #1/#2, the user's literal ask)** — enlarges/launders the SAME author's selection; does NOT sever curator-from-worker; spends a shipped-framework secrets/context/dilution price (C) + leans toward synthesis the sonnet tier was cut against (ADR 0006). Abridged is worst (false-confidence halo).
- **Independent extractor agent reading a RAW transcript (A's #3 — the only full cure)** — severs curator-from-worker for ALL perception failures, but gated on an unproven raw-transcript-channel feasibility AND re-opens ADR 0006. Deferred to the gated trigger; the git cross-check is the affordable down payment.
