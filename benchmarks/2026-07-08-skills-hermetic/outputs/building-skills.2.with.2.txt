# JSON Config Check

## Validate syntax

Parse the file with Python's `json` module. A `json.JSONDecodeError` gives the exact line, column, and reason — report that directly rather than a generic "invalid JSON" message.

```python
import json

with open(config_path) as f:
    try:
        data = json.load(f)
    except json.JSONDecodeError as e:
        print(f"{config_path}:{e.lineno}:{e.colno}: {e.msg}")
        raise
```

## Validate against schema (if present)

Look for a schema file named `<config_name>.schema.json` in the same directory, or a `$schema` key inside the config itself. If found, validate with `jsonschema`:

```python
import jsonschema

jsonschema.validate(instance=data, schema=schema)
```

Report each validation error's `.message` and `.absolute_path` so the user can locate the offending field. If no schema is found, skip this step — syntax validation alone is still useful.

## Report results

- Valid, no schema: "Valid JSON."
- Valid, schema passed: "Valid JSON, matches schema."
- Syntax error: file, line, column, and reason.
- Schema violation: field path and the specific rule violated.
```
