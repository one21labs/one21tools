# CLAUDE.md — checkout-service

## Stack

- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Prisma on Postgres

## History & Ownership

Started as a Rails prototype in 2021, fully rewritten in 2023 by the platform team.
Currently owned by the **payments squad**.

## Money Handling — Critical Rule

**All monetary amounts are integer cents. Floats for money are banned, no exceptions.**

In 2024, a float-rounding bug corrupted invoices in production. Any code that
represents money as a float (including intermediate calculations, API payloads,
or DB columns) is a bug. Always use integer cents.

## Testing

- `npm test` — runs the Playwright E2E suite. **Requires a running dev server**;
  it will fail if one isn't up. Start the dev server first (`npm run dev`) before
  running this.
- `npm run test:unit` — unit tests, no dev server required.

## Database Migrations

**Always use `./scripts/migrate.sh` for migrations.**

Do not run `prisma migrate` directly — it breaks the shadow database that CI
relies on. This applies to all migration operations (create, apply, reset, etc.).

## Auth

API routes authenticate using a custom `X-Session-Token` header scheme — **not**
standard `Authorization: Bearer` tokens. Full scheme details are documented in
`docs/api.md`. Consult that doc before implementing or modifying auth logic.

## Code Style

Full style guide: `docs/style-guide.md` (45 pages — consult it for anything not
covered here).

The only deviations from typical TypeScript style are:

- **No default exports** — use named exports everywhere.
- **Zod schemas are colocated** next to their route handlers (not in a separate
  schemas directory).

## Deploy Procedure

Identical across all six company repos:

1. Merge to `main`
2. Wait for CI to go green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`
