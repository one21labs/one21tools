# CLAUDE.md – checkout-service

## Stack
Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

## Money handling
**All amounts are integer cents; floats for money are banned.** A float-rounding incident in 2024 corrupted invoices—this is a hard rule, not a style preference. If you find or add code using floats for money, fix it immediately.

## Testing
- `npm test` — runs Playwright E2E suite **and requires a running dev server** (`npm run dev` in another terminal first). Fails silently without it.
- `npm run test:unit` — fast, standalone unit tests.

## Migrations
Never run `prisma migrate` directly; always use `./scripts/migrate.sh`. Running Prisma directly breaks the shadow database that CI relies on.

## Auth
API routes authenticate via a custom `X-Session-Token` header scheme (not standard Bearer tokens). The full scheme is documented in `docs/api.md`—do not assume Bearer or standard JWT patterns.

## Style
Two deviations from typical TypeScript conventions:
- No default exports.
- Zod schemas colocated next to their route handlers.

For the rest, see `docs/style-guide.md` (45 pages).

## Deploys
All six company repos use the same standard deployment procedure:
1. Merge to main.
2. Wait for CI to pass.
3. Run `npm run build:prod`.
4. Run `./deploy staging`.
5. Smoke-test `/healthz` endpoint.
6. Page on-call if p95 latency exceeds 500ms.
7. Run `./deploy prod`.
8. Announce in #releases.

No repo-specific deviations.
