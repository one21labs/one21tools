**Suggested requirements**
- Minimum 12 characters
- At least one uppercase, one lowercase, one digit, one special character
- Reject common/breached passwords (top-10k list or HaveIBeenPwned k-anonymity API)
- Reject passwords containing the user's email/username
- No leading/trailing whitespace-only padding tricks

**Validator (JS, framework-agnostic — use on both client and server; client-side alone is not real security)**

```js
const COMMON_PASSWORDS = new Set([
  "password", "123456", "12345678", "qwerty", "letmein", "111111",
  "iloveyou", "admin", "welcome", "password1"
  // load a real top-10k list in production
]);

function validatePasswordStrength(password, { email = "", username = "" } = {}) {
  const errors = [];

  if (typeof password !== "string" || password.length < 12) {
    errors.push("Password must be at least 12 characters long.");
  }
  if (!/[a-z]/.test(password)) errors.push("Password must contain a lowercase letter.");
  if (!/[A-Z]/.test(password)) errors.push("Password must contain an uppercase letter.");
  if (!/[0-9]/.test(password)) errors.push("Password must contain a digit.");
  if (!/[^A-Za-z0-9]/.test(password)) errors.push("Password must contain a special character.");

  const lower = password.toLowerCase();
  if (COMMON_PASSWORDS.has(lower)) {
    errors.push("This password is too common.");
  }
  if (email && lower.includes(email.split("@")[0].toLowerCase())) {
    errors.push("Password must not contain your email address.");
  }
  if (username && lower.includes(username.toLowerCase())) {
    errors.push("Password must not contain your username.");
  }

  return { valid: errors.length === 0, errors };
}

module.exports = { validatePasswordStrength };
```

**Registration form hook (React example)**

```jsx
const [password, setPassword] = useState("");
const [errors, setErrors] = useState([]);

function handleSubmit(e) {
  e.preventDefault();
  const { valid, errors } = validatePasswordStrength(password, { email });
  if (!valid) {
    setErrors(errors);
    return;
  }
  // submit to server — server MUST re-validate, never trust client-only checks
}
```

**Server-side note:** always re-run this same check server-side before hashing (bcrypt/argon2) and storing — client validation is UX only, not enforcement.
