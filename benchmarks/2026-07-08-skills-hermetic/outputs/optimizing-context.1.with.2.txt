```markdown
# checkout-service

Payments squad's checkout service. Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres.

Originally a 2021 Rails prototype, rewritten by the platform team in 2023. Now owned by the payments squad — expect some legacy naming/structure from the rewrite.

Onboarding is fully covered in README.md — don't duplicate it here.

## Money handling

All monetary amounts are **integer cents**. Never use floats for money — a 2024 float-rounding bug corrupted invoices. If you see a float touching a money value, treat it as a bug.

## Tests

- `npm test` runs the **Playwright E2E suite** — it requires a running dev server and will fail without one. Don't reach for it as a quick sanity check.
- `npm run test:unit` runs unit tests — use this for fast, isolated checks.

## Database migrations

Always run migrations through `./scripts/migrate.sh`. Never run `prisma migrate` directly — it breaks the shadow database CI relies on.

## Auth

API routes authenticate via a custom `X-Session-Token` header scheme — **not** standard Bearer tokens. Full scheme documented in `docs/api.md`; check it before implementing or debugging auth-related routes.

## Style

Style guide lives at `docs/style-guide.md`. Two deviations from typical TypeScript conventions to know without reading all 45 pages:
- No default exports.
- Zod schemas are colocated next to their route handlers.

## Deploys

Same procedure as the other five company repos:
1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. Page on-call if p95 latency exceeds 500ms.
7. `./deploy prod`.
8. Announce in `#releases`.

Treat steps 7-8 as production-affecting — confirm with the user before executing them rather than running the sequence unattended.
```
