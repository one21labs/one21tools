**Database-level (Postgres CHECK constraint — best if drift must never happen):**
```sql
ALTER TABLE users
ADD CONSTRAINT name_length_matches_full_name
CHECK (name_length = char_length(full_name));
```
This rejects any insert/update where the two go out of sync, so the bug can't recur regardless of which code path writes the record.

**Application-level validator (Python example, adapt to your language):**
