# JSON Config Check

## Overview

This skill validates JSON configuration files. It parses a file for syntax errors and optionally validates the parsed JSON against a schema file if one exists in the same directory.

## Inputs

- **config_file**: Path to the JSON configuration file to validate
- **schema_file** (optional): Path to a JSON schema file in the same directory as the config

## Procedure

1. Read the configuration file
2. Parse it as JSON using Python's `json` module
3. If parsing fails, report the syntax error with line number and stop
4. If a JSON schema file exists in the same directory, validate the parsed JSON against it using the `jsonschema` package
5. If schema validation fails, report validation errors
6. If all checks pass, report success

## When to Use

- Verify a JSON file is syntactically valid before deployment
- Check that a JSON configuration matches an expected schema
- Catch JSON formatting errors early in the development cycle
