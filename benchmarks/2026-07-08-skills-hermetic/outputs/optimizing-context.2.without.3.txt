# Project Guidelines

## About This Project
Internal orders dashboard built with Flask (Python 3.11).

## Code Style
- 4 spaces indentation (PEP 8)
- Meaningful variable names
- Docstrings for public functions
- Keep functions short and focused
- Prefer f-strings over `.format()`

## Getting Started
See README.md for setup steps.

## Database
Orders table schema is at version 12 (defined in `models.py`). Never use raw SQL.

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere — a 2023 bug from naive datetimes double-charged customers in CET.
- Run `make lint` before committing; CI runs the same target.
- Local tests: `pytest -m "not integration"`; the integration marker requires the docker compose stack running.
