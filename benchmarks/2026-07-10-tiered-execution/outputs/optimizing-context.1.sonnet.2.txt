```markdown
# CLAUDE.md

Guidance for Claude Code when working in this repo.

## Project

`checkout-service` — Next.js 14 (App Router), TypeScript, Tailwind, Prisma on
Postgres. Owned by the payments squad. Originally a 2021 Rails prototype,
rewritten from scratch by the platform team in 2023 — don't expect Rails
conventions or artifacts to still be relevant.

Onboarding (clone/install/env/dev server) is covered in README.md — don't
duplicate it here, just read that if you need setup steps.

## Money handling — hard rule

**All monetary amounts are integer cents. Never use floats for money.**

A float-rounding bug in 2024 corrupted invoices in production. This is a
zero-tolerance rule, not a style preference — if you see or are about to
introduce a `number`/float being used for a currency amount, stop and fix it
to integer cents instead.

## Tests

- `npm test` — runs the Playwright E2E suite. Requires a running dev server;
  it will fail if one isn't already up. Don't treat that failure as a code
  bug without first checking whether the dev server is running.
- `npm run test:unit` — unit tests, no server required.

## Database migrations

Always run migrations via `./scripts/migrate.sh`. **Never run `prisma
migrate` directly** — it breaks the shadow database CI depends on.

## Auth

API routes authenticate via a custom `X-Session-Token` header scheme — this
is **not** standard Bearer/OAuth. Full scheme details are in `docs/api.md`;
consult it before implementing or debugging auth-related code.

## Code style

Full style guide: `docs/style-guide.md` (45 pages — consult it for anything
not covered below). Deviations from typical TypeScript style:

- No default exports — use named exports everywhere.
- Zod schemas are colocated next to the route handlers that use them, not in
  a shared schemas directory.

## Deploys

Same procedure as the other five repos in the company. Do not perform any of
these steps autonomously without explicit user confirmation — this affects
shared production systems:

1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`
```

A couple of notes on choices I made:
- I deliberately left onboarding out (README already owns it) to avoid duplication that could drift out of sync.
- I flagged the deploy steps as requiring explicit confirmation since several are irreversible/production-affecting — adjust if your team wants Claude Code to run further unattended.
