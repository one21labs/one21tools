```markdown
# Sanitizing Customer Exports

Redacts PII from customer CSV exports so they're safe to share outside the
company. The rules below are fixed — they are not a matter of judgment per
request, so apply them exactly the same way every time.

## Fixed rules

1. **Drop columns**: `email`, `phone`, `ssn` (case-insensitive, exact
   column-name match) are removed entirely from the output. Their values
   never appear in the redacted file.
2. **Replace name**: a column named `name` (case-insensitive) is replaced
   with an opaque per-row id (`CUST-000001`, `CUST-000002`, ...). The id is
   sequential within the file, not derived from the original name, and is
   not guaranteed stable across different export runs.
3. **Mask free text**: free-text columns — `notes` and `comments` by
   default — are scanned for embedded email addresses and phone numbers.
   Matches are replaced in place with `[EMAIL_REDACTED]` /
   `[PHONE_REDACTED]`; the rest of the text is left untouched.
4. **Never touch the original**: the source file is opened read-only. The
   redacted copy is always written to a new file, `<original>.redacted.csv`
   (e.g. `customers.csv` → `customers.redacted.csv`), next to the input
   file. If the redaction script ever cannot guarantee it's writing to a
   different path than the input, it aborts rather than risk an overwrite.

If a CSV doesn't have one of these columns, that rule is simply a no-op for
that column — don't treat a missing column as an error.

## Workflow

1. Confirm the exact path to the source CSV with the user if it's not
   already given.
2. Run the redaction script:
   ```
   python3 scripts/redact_csv.py <path/to/export.csv>
   ```
   If the export uses different free-text column names than `notes`/
   `comments`, pass them explicitly:
   ```
   python3 scripts/redact_csv.py <path/to/export.csv> --free-text-columns notes,comments,description
   ```
3. The script prints the output path and a summary of what it dropped,
   replaced, and scanned. Report that summary back to the user so they can
   confirm the redaction matched their expectations before sharing the
   file.
4. Before telling the user the file is ready to share, spot-check the
   output yourself (open the first few rows) to confirm:
   - none of `email`, `phone`, `ssn` remain as columns
   - the name column now shows `CUST-######` values
   - any emails/phone numbers that were embedded in notes/comments are now
     masked
   - the original input file is unchanged (same size/mtime as before)

## Known limitations

See [reference/pii-patterns.md](reference/pii-patterns.md) for the exact
regex patterns used and their gaps (e.g. it does not scan free text for
SSNs or names, and international phone formats may not all match). If a
customer export has PII shapes beyond email/phone in free text (e.g. street
addresses, SSNs written in notes), flag that explicitly to the user rather
than silently treating the file as fully clean — this skill's masking is
scoped only to emails and phone numbers embedded in free text, per the
fixed rules above.

## Example

See `examples/sample_input.csv` and `examples/sample_output.redacted.csv`
for a worked before/after example.
```

```python
#!/usr/bin/env python3
"""
Redact PII from a customer CSV export before it is shared outside the company.

Fixed rules (not configurable beyond --free-text-columns):
  1. Columns named email, phone, ssn (case-insensitive, exact match) are
     dropped from the output entirely.
  2. A column named name (case-insensitive) has its values replaced with
     an opaque per-row id: CUST-000001, CUST-000002, ...
  3. Free-text columns (default: notes, comments; case-insensitive) are
     scanned for embedded email addresses and phone numbers, which are
     masked in place with [EMAIL_REDACTED] / [PHONE_REDACTED]. The rest
     of the free-text value is left untouched.
  4. The original file is opened read-only. Output is written to a new
     file: <original-basename>.redacted.csv, next to the input file.
     If the input doesn't end in .csv, ".redacted.csv" is appended to
     the full filename instead.

Usage:
  python3 redact_csv.py customers.csv
  python3 redact_csv.py customers.csv --free-text-columns notes,comments,description
"""

import argparse
import csv
import os
import re
import sys

DROP_COLUMNS = {"email", "phone", "ssn"}
NAME_COLUMN = "name"
DEFAULT_FREE_TEXT_COLUMNS = {"notes", "comments"}

EMAIL_RE = re.compile(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}")
PHONE_RE = re.compile(
    r"(?<!\d)(?:\+?1[-.\s]?)?\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}(?!\d)"
)


def mask_free_text(value):
    if not value:
        return value
    value = EMAIL_RE.sub("[EMAIL_REDACTED]", value)
    value = PHONE_RE.sub("[PHONE_REDACTED]", value)
    return value


def output_path_for(input_path):
    base, ext = os.path.splitext(input_path)
    if ext.lower() == ".csv":
        return f"{base}.redacted.csv"
    return f"{input_path}.redacted.csv"


def redact_csv(input_path, free_text_columns):
    if not os.path.isfile(input_path):
        sys.exit(f"error: input file not found: {input_path}")

    out_path = output_path_for(input_path)
    if os.path.abspath(out_path) == os.path.abspath(input_path):
        sys.exit("error: computed output path matches input path; aborting to avoid overwrite")

    with open(input_path, "r", newline="", encoding="utf-8") as infile:
        reader = csv.DictReader(infile)
        if reader.fieldnames is None:
            sys.exit("error: input CSV has no header row")

        fieldnames = reader.fieldnames
        lower_to_original = {f.lower(): f for f in fieldnames}

        drop = {lower_to_original[c] for c in DROP_COLUMNS if c in lower_to_original}
        name_col = lower_to_original.get(NAME_COLUMN)
        free_text_cols = {
            lower_to_original[c]
            for c in free_text_columns
            if c in lower_to_original
        }

        out_fieldnames = [f for f in fieldnames if f not in drop]

        rows = []
        counter = 0
        for row in reader:
            counter += 1
            out_row = {}
            for field in out_fieldnames:
                value = row.get(field, "")
                if field == name_col:
                    value = f"CUST-{counter:06d}"
                elif field in free_text_cols:
                    value = mask_free_text(value)
                out_row[field] = value
            rows.append(out_row)

    with open(out_path, "w", newline="", encoding="utf-8") as outfile:
        writer = csv.DictWriter(outfile, fieldnames=out_fieldnames)
        writer.writeheader()
        writer.writerows(rows)

    return out_path, len(rows), sorted(drop), name_col, sorted(free_text_cols)


def main():
    parser = argparse.ArgumentParser(description="Redact PII from a customer CSV export.")
    parser.add_argument("input_csv", help="Path to the source CSV export")
    parser.add_argument(
        "--free-text-columns",
        default=",".join(sorted(DEFAULT_FREE_TEXT_COLUMNS)),
        help="Comma-separated list of free-text columns to scan for embedded "
             "emails/phone numbers (default: notes,comments)",
    )
    args = parser.parse_args()

    free_text_columns = {c.strip().lower() for c in args.free_text_columns.split(",") if c.strip()}

    out_path, row_count, dropped, name_col, scanned = redact_csv(args.input_csv, free_text_columns)

    print(f"wrote {out_path} ({row_count} rows)")
    print(f"dropped columns: {dropped if dropped else '(none found)'}")
    print(f"name column replaced with opaque id: {name_col if name_col else '(none found)'}")
    print(f"free-text columns scanned for embedded PII: {scanned if scanned else '(none found)'}")


if __name__ == "__main__":
    main()
```

```markdown
# PII detection patterns and limitations

## Email regex
`[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}`

Covers standard `local@domain.tld` addresses. Does not handle quoted-string
local parts or internationalized domain names (punycode addresses will
mostly still match if ASCII).

## Phone regex
`(?<!\d)(?:\+?1[-.\s]?)?\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}(?!\d)`

Matches common US/Canada 10-digit formats with an optional leading `+1`:
`555-123-4567`, `(555) 123-4567`, `555.123.4567`, `+1 555 123 4567`,
`5551234567`. Does not reliably match non-NANP international numbers
(e.g. UK, EU formats) — extend the pattern if an export contains those.

## Explicitly out of scope for this skill

- Free-text columns are only scanned for **emails and phone numbers**, per
  the fixed rules. SSNs, street addresses, dates of birth, or names
  appearing inside notes/comments are **not** detected or masked. If a
  customer export is known to contain these in free text, tell the user
  before sharing the "redacted" file — it is not fully scrubbed.
- The opaque id replacing `name` is a per-file sequential counter
  (`CUST-000001`, ...). It is not a stable hash — running the script twice
  on the same input, or on two different exports referencing the same
  customer, will not produce the same id. If cross-export joins on a
  stable pseudonymous id are needed, this script would need to be extended
  (e.g. a salted hash of the original name persisted in a lookup table) —
  don't assume that behavior exists today.
- Column matching is exact-name, case-insensitive (`Email`, `EMAIL`,
  `email` all match `email`). A column like `Email_Address` or `work_email`
  will **not** be dropped automatically — check the header row of any new
  export format before running the script.
```
