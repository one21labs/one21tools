```markdown
# Project Guidelines

## Overview
Internal orders dashboard built with Flask. See README.md for setup and installation instructions.

## Code Style
- Python 3.11 with 4-space indentation (PEP 8)
- Use meaningful variable names
- Write docstrings for public functions
- Keep functions short and focused
- Prefer f-strings over `.format()`

## Database
- Orders table schema is at version 12 (defined in models.py)
- Never use raw SQL; use ORM only

## Development Conventions
- Use `datetime.now(tz=timezone.utc)` everywhere (naive datetimes caused a 2023 double-charging bug in CET)
- Run `make lint` before committing (CI runs the same target)
- Local tests: `pytest -m "not integration"`; integration tests require the docker compose stack up
```
