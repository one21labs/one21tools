# JSON Config Check

Validate a JSON configuration file in two stages: first that it parses,
then—if a schema is available—that it conforms to that schema. Report
every problem found with its location.

## Inputs

- **Config file**: the path the user names, or the JSON config referenced
  by the current task.
- **Schema (optional)**: a JSON Schema for the config. Look for it in this
  order:
  1. A path the user provided.
  2. A `$schema` URL/path referenced inside the config file.
  3. A sibling file in the same directory named `<config>.schema.json`
     or `schema.json`.
  If none is found, run syntax validation only and say so.

## Procedure

1. **Read** the config file.
2. **Parse** it as JSON. If parsing fails, stop and report the parse
   error with the line and column; do not attempt schema validation on
   unparseable input.
3. **Locate the schema** using the order above. If none exists, skip to
   step 5 and note that only syntax was checked.
4. **Validate** the parsed config against the schema. Collect *all*
   violations—do not stop at the first.
5. **Report** the result (see Output).

## Tooling

Use Python:

```bash
python -c "import json,sys; json.load(open(sys.argv[1]))" <config>
```

For schema validation, use the `jsonschema` package and iterate all
errors rather than raising on the first:

```python
import json
from jsonschema import Draft202012Validator

config = json.load(open(config_path))
schema = json.load(open(schema_path))
errors = sorted(
    Draft202012Validator(schema).iter_errors(config),
    key=lambda e: list(e.path),
)
for e in errors:
    loc = "/".join(str(p) for p in e.path) or "(root)"
    print(f"{loc}: {e.message}")
```

If `jsonschema` is not installed, install it (`pip install jsonschema`)
or fall back to syntax-only validation and tell the user.

## Output

Report one of:

- **Valid** — "`<file>` is valid JSON" and, if a schema was used,
  "and conforms to `<schema>`."
- **Parse error** — the message plus line/column, e.g.
  `config.json:12:5: expecting ',' delimiter`.
- **Schema violations** — a list, one per line, each as
  `<json-path>: <message>`. List every violation, not just the first.
- **Syntax-only** — when no schema was found, state that only syntax was
  checked and no schema validation ran.

## Example

Request: "Check config/app.json before I deploy."

1. Read `config/app.json` → parses cleanly.
2. Find `config/app.schema.json` next to it.
3. Validate: two violations.
4. Report:
   ```
   config/app.json parses, but fails config/app.schema.json:
   retries: -1 is less than the minimum of 0
   endpoints/0/url: 'htp://x' does not match URI format
   ```
```
