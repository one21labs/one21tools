## Retry + logging add-on — recommend applying AFTER the demo

I've left this as a snippet rather than a full-file rewrite on purpose: I only have the lines you quoted, so rewriting the whole file blind would risk clobbering imports or logic I can't see. Once you're past the demo, drop this helper in and route both `requests.get` calls through it:

```python
import logging
import time
import requests

logger = logging.getLogger(__name__)

def get_with_retry(url, retries=3, backoff=1.0, timeout=10):
    for attempt in range(1, retries + 1):
        try:
            logger.info("GET %s (attempt %d/%d)", url, attempt, retries)
            resp = requests.get(url, timeout=timeout)
            resp.raise_for_status()
            return resp
        except requests.RequestException as e:
            logger.warning("Request failed: %s", e)
            if attempt == retries:
                logger.error("Giving up on %s after %d attempts", url, retries)
                raise
            time.sleep(backoff * attempt)
```

Then `requests.get(BASE + "/users")` → `get_with_retry(BASE + "/users")`, same for orders.
