```markdown
# JSON Config Check

## Purpose

This skill validates JSON configuration files for syntax errors and structural correctness. When a JSON Schema is present in the same directory, it also validates the configuration against the schema to ensure required fields and types match.

## When to Use

- Validating configuration files before deployment
- Checking for malformed JSON (syntax errors)
- Verifying configuration structure matches expected schema
- Identifying schema violations (missing required fields, type mismatches)

## Inputs

- **config_file**: Path to the JSON configuration file to validate
- **schema_file** (optional): Path to a JSON Schema file in the same directory. If present, the config is validated against this schema.

## Outputs

- **valid**: Boolean indicating whether the JSON is well-formed and (if applicable) matches the schema
- **errors**: List of validation errors (empty if valid). Each error includes line number, message, and type (syntax or schema validation)

## How It Works

1. Parse the JSON configuration file for syntax validity
2. If the file parses successfully and a schema file exists in the same directory, validate the configuration against the schema
3. Report any errors found, or confirm validity

## Error Scenarios

- **Syntax Error**: Malformed JSON (unclosed brackets, invalid escaping, trailing commas)
- **Schema Violation**: Valid JSON that doesn't match the provided schema (missing required fields, incorrect types, constraint violations)
- **Missing Config File**: Configuration file does not exist or is not readable
- **Invalid Schema**: Schema file exists but is malformed or invalid

## Implementation

Uses Python's `json` module for parsing and the `jsonschema` package for schema validation.
```
