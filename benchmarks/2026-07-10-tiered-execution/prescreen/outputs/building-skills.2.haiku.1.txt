```markdown
# JSON Config Check

## Purpose

This skill validates JSON configuration files, checking for valid syntax and conformance to a provided JSON schema. Use it to catch configuration errors before deployment.

## When to Use

- Validating configuration files in your project
- Checking JSON syntax before committing
- Verifying configuration against schema requirements
- Detecting missing or invalid fields in JSON configs

## How to Use

Provide the path to a JSON configuration file and an optional JSON schema file. The skill will:

1. Parse the JSON file for valid syntax
2. Validate against the schema if provided
3. Report any errors found

### Parameters

- **config_file** (required): Path to the JSON configuration file to validate
- **schema_file** (optional): Path to the JSON schema file for validation

### Output

Returns validation results:
- **valid**: Boolean indicating if validation passed
- **errors**: Array of error messages (syntax or schema violations)
- **summary**: Brief description of validation outcome

### Example

If your config file at `config.json` has a syntax error or missing required field defined in `schema.json`, the skill will report the specific error location and type.
```
