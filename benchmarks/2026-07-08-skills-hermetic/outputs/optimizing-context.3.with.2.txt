```
6. User-level CLAUDE.md / user preferences — this is Dana's personal habit that should follow her across every project, not a repo-specific rule.

7. None of the standard mechanisms — this is a fast-changing fact fully owned by the code itself (the migrations folder / a schema_version table). (If you want zero-lookup-cost convenience, the only safe compromise is a one-line pointer in repo CLAUDE.md to *where* to look — never the value itself.)

8. Plain one-off prompt — a single, ephemeral, non-repeating task for today doesn't warrant persistence in any mechanism.
```
