# checkout-service

Next.js 14 (App Router) + TypeScript + Tailwind + Prisma/Postgres checkout service, owned by the payments squad.

## Key Decisions

- All money amounts are integer cents. Never use floats for money — a 2024 float-rounding bug corrupted invoices.
- API routes authenticate via a custom `X-Session-Token` header, not standard Bearer tokens. Full scheme: docs/api.md.
- No default exports; Zod schemas are colocated next to their route handlers. Full style guide: docs/style-guide.md.

## Common Mistakes

- `npm test` runs the Playwright E2E suite and requires a running dev server — it will fail otherwise. For unit tests use `npm run test:unit`.
- Never run `prisma migrate` directly — it breaks the shadow database CI relies on. Always use `./scripts/migrate.sh`.

## Deploy

Same procedure used across all six company repos:
1. Merge to main, wait for CI green
2. `npm run build:prod`
3. `./deploy staging`
4. Smoke-test `/healthz`
5. Page on-call if p95 latency > 500ms
6. `./deploy prod`
7. Announce in #releases

## Where to Look

- Onboarding: README.md (already covers clone/install/env/dev)
- Auth scheme details: docs/api.md
- Style guide: docs/style-guide.md
