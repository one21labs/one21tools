Here's the fixed module. The only change is wrapping the per-line parse in a `try/except` so a bad line gets skipped instead of aborting the batch:

```python
# price_feed.py

def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, TypeError):
            # Malformed line (wrong field count, non-numeric price, etc.) -- skip it
            # so one bad row can't abort the whole batch. Clean up properly next sprint.
            continue
    return prices
```

What this catches, without touching anything else:
- **Wrong field count** — `"a,b,c"` or `"garbage"` → `line.split(",")` raises `ValueError` on unpack → skipped.
- **Non-numeric price** — `"SKU1,abc"` → `float()` raises `ValueError` → skipped.
- **`None`/bad type in the list** → `TypeError` → skipped.

Good rows in the same batch still apply normally.
