## CLAUDE.md

```markdown
# Atlas Monorepo — Claude Code Conventions & Setup

## Monorepo Structure
- **api/** — Go/gRPC backend services (protobuf-driven)
- **web/** — React/TypeScript frontend
- **pipeline/** — Python DAGs (Airflow orchestration)

Each service has its own `CLAUDE.md` with language-specific guidance. Start there when working in a service.

## Org-Wide Conventions (All Teams)

### Commits & Branching
- **Conventional Commits:** Follow `type(scope): description` (e.g., `fix(auth): handle expired tokens`). `type` must be one of: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`, `ci`.
- **Trunk-Based Development:** All work flows through `main`. No long-lived feature branches. Push small, reviewed PRs; land them quickly.
- **No Force Push:** Never force-push to `main` or any merged branch.

### CI/CD & Quality Gates
- **`make check` Must Pass:** Before opening any PR, run `make check` locally. This is a hard requirement; CI will block PRs that fail it.
- **make check** runs linting, unit tests, type checks, and basic correctness checks across all services.

### Code Review & Testing
- All PRs require approval before landing.
- Write tests that catch regressions; coverage expectations vary by service (see service-specific CLAUDE.md).

---

## Critical Quirks & Gotchas

### api/gen/ — Generated Code
**Everything under `api/gen/` is generated by `make proto`. Do not edit it by hand.** Hand edits will be silently overwritten on the next `make proto` run.
- To change generated code, edit the `.proto` files, then run `make proto`.
- See `/api/CLAUDE.md` for proto best practices.

### web/ — Playwright Test Modes
- `npm test` runs the full Playwright integration suite (opens real browsers, slower).
- `npm run test:unit` runs Jest unit tests only (no browser, fast).
- Use `npm run test:unit` for quick feedback; use `npm test` before PR.

### pipeline/ — AIRFLOW_HOME Setup
**DAGs will fail to import unless `AIRFLOW_HOME` is set to `pipeline/.airflow` first.**
```bash
export AIRFLOW_HOME=pipeline/.airflow
# Then run tests or local dev
```
Without this, DAG import errors are cryptic and easy to misdiagnose. Always check AIRFLOW_HOME first.

## General Principles

1. **Trust the build.** If `make check` passes, the code is safe to merge.
2. **Fail fast.** Run `make check` before opening a PR; don't wait for CI.
3. **One service at a time.** Focus on one service per task unless the task explicitly spans services.
4. **Generated code is off-limits.** If it's in `api/gen/`, you don't edit it; you edit the source (proto files).
5. **Test locally before PR.** Each service has a test story; use it.

### Key Design Decisions

1. **Root CLAUDE.md bridges all services** — Org-wide conventions live at the root so trunk-based dev, conventional commits, and `make check` are universally visible.

2. **Service-specific docs isolate quirks** — Each service gets its own CLAUDE.md, so Go engineers see gRPC gotchas, React engineers see Playwright quirks, etc. Claude will find the right doc when contextualized to a service directory.

3. **Incident runbook and warehouse SQL are centralized** — These are org-wide resources that transcend service boundaries. They live in `.claude/instructions/` and are referenced from the root so they're always accessible.

4. **Generated code gets a hard warning in root** — api/gen/ is the most dangerous foot-gun; it's called out explicitly so Claude never accidentally edits it.

5. **Test quirks are delegated to service docs** — npm test vs. npm run test:unit, Playwright, AIRFLOW_HOME—each is documented where it matters, with brief pointers from the root.
