# CLAUDE.md

Instructions for Claude Code when working in checkout-service.

## Project Overview

Next.js 14 (App Router) + TypeScript + Tailwind + Prisma on Postgres. Owned by the payments squad. (Originally a 2021 Rails prototype, rewritten in TypeScript by the platform team in 2023 — no Rails code remains.)

Onboarding is covered in README.md (clone, `npm install`, copy `.env.example` to `.env`, `npm run dev`) — follow that, not this file, for local setup.

## Critical Rule: Money

All monetary amounts are **integer cents**. Never use floats for money, anywhere — arithmetic, storage, API payloads, or display formatting. A float-rounding bug corrupted invoices in 2024. Convert to display currency only at the final presentation layer.

## Database Migrations

Always run migrations through `./scripts/migrate.sh`. Never run `prisma migrate` directly — it breaks the shadow database that CI depends on.

## Testing

- `npm test` — Playwright E2E suite. Requires a running dev server; will fail otherwise.
- `npm run test:unit` — unit tests, no dev server needed.

## Authentication

API routes use a custom `X-Session-Token` header scheme, not standard Bearer/OAuth tokens. Full scheme documented in `docs/api.md` — consult it before implementing or modifying auth-related code.

## Code Style

Full style guide: `docs/style-guide.md`. Two deviations from typical TypeScript conventions to always apply:
- No default exports — named exports only.
- Zod schemas are colocated next to their route handlers, not in a separate schemas directory.

## Deploy Procedure

Same procedure across all six company repos:

1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`

Do not skip staging or the smoke test, even for small changes.
