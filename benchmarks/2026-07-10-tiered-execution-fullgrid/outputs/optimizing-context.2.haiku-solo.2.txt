```markdown
# Project Guidelines

## Stack
Python 3.11 Flask web application serving an internal orders dashboard. See README.md for setup.

## Code Style
- 4 spaces for indentation (PEP 8)
- Meaningful variable names
- Docstrings for public functions
- Prefer f-strings over .format()

## Database
- Orders table schema is at version 12 (models.py)
- Never use raw SQL

## Critical Conventions
- **Timezones**: Always use `datetime.now(tz=timezone.utc)`. A 2023 bug from naive datetimes double-charged customers in CET zones.
- **Pre-commit**: Run `make lint` (CI runs the same target)
- **Testing locally**: `pytest -m "not integration"`; integration tests require the docker compose stack to be running
```
