# JSON Config Check

Validates a JSON configuration file for syntax errors and, if a schema is available, checks it against that schema.

## Steps

1. Read the target JSON file.
2. Parse it with Python's `json` module (`json.load`). If parsing fails, report the exact error message and
   line/column from the `json.JSONDecodeError`, then stop.
3. Look for a schema to validate against, in this order:
   - A `$schema` key inside the JSON file, resolved relative to the file's directory.
   - A sibling file with the same base name and a `.schema.json` suffix (e.g. `config.json` → `config.schema.json`).
   If no schema is found, report that the file is syntactically valid JSON and stop — schema validation is
   optional, not required.
4. If a schema is found, validate the parsed JSON against it with the `jsonschema` package (`jsonschema.validate`).
5. Report results:
   - Valid: confirm the file is valid JSON and, if applicable, conforms to the schema.
   - Invalid: report every validation error found (not just the first), including the JSON path to the
     offending field and the reason it failed.

## Example

​```python
import json
import jsonschema

with open("config.json") as f:
    data = json.load(f)  # raises JSONDecodeError on malformed JSON

with open("config.schema.json") as f:
    schema = json.load(f)

jsonschema.validate(instance=data, schema=schema)  # raises ValidationError on mismatch
​```

## Out of scope

This skill only checks syntax and schema compliance. It does not check business-logic correctness of config
values, does not modify files, and does not generate a schema for files that lack one.
```
