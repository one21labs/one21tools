# JSON Config Check

Validate a JSON configuration file, and — when a schema is available — check it
against that schema. Report every error found in a single pass.

## When to use

- The user asks to validate, check, or lint a JSON config file.
- You are editing a JSON config that has a sibling schema file.
- A config change is about to be committed or deployed.

## Inputs

- **Config file** (required): path to the JSON file to validate.
- **Schema file** (optional): a JSON Schema. If not given, look for one next to
  the config: `<name>.schema.json`, `schema.json`, or a `$schema` URL/path
  referenced inside the config itself.

## Procedure

1. **Read and parse the config.** Load the file and parse it as JSON. If parsing
   fails, stop and report the syntax error with its line and column — do not
   attempt schema validation on unparseable input.

2. **Locate the schema.** Use the schema path if provided; otherwise search the
   config's directory for the candidates listed above. If no schema is found,
   report that the JSON is syntactically valid but was not schema-checked, and
   note which filenames you looked for.

3. **Validate against the schema.** Prefer Python:

   ```bash
   python3 - "$CONFIG" "$SCHEMA" <<'PY'
   import json, sys
   from jsonschema import Draft202012Validator
   config = json.load(open(sys.argv[1]))
   schema = json.load(open(sys.argv[2]))
   errors = sorted(Draft202012Validator(schema).iter_errors(config),
                   key=lambda e: list(e.path))
   for e in errors:
       loc = "/".join(map(str, e.path)) or "(root)"
       print(f"{loc}: {e.message}")
   print(f"\n{len(errors)} error(s)")
   PY
   ```

   If `jsonschema` is not installed, install it (`pip install jsonschema`) or,
   if that isn't possible, fall back to `check-jsonschema` or `ajv`. State which
   tool you used.

4. **Collect all errors.** Report every violation found, not just the first —
   use `iter_errors` (above) rather than `validate`, which stops at the first
   failure.

## Output

Report, in this order:

- **Result**: `valid`, `invalid`, or `valid (no schema found)`.
- **Errors**: for each, the JSON path to the offending field and a one-line
  message. For syntax errors, include line and column.
- **Summary**: total error count.

Keep the report concise — path, message, count. Do not restate the file's
contents.
```
