# Claude.md - checkout-service

## Overview
Checkout service for the payments squad. Next.js 14 (App Router) with TypeScript, Tailwind, Prisma on Postgres.

**Onboarding**: See README.md for full setup (clone → `npm install` → copy `.env.example` to `.env` → `npm run dev`).

## Critical Constraints

### Money Handling
**All amounts are integer cents.** Floats are banned. A float-rounding incident in 2024 corrupted invoices — do not use floats for currency under any circumstances.

### Migrations
**Always use `./scripts/migrate.sh`** — never run `prisma migrate` directly. Direct migrations break the shadow database our CI relies on.

### Testing
- **E2E tests**: `npm test` (runs Playwright suite; requires running dev server)
- **Unit tests**: `npm run test:unit`

### Authentication
API routes use a custom `X-Session-Token` header scheme (not standard Bearer tokens). Full documentation: `docs/api.md`.

## Code Style & Architecture

See `docs/style-guide.md` (45 pages).

**Key deviations from standard TypeScript style:**
- No default exports
- Zod schemas colocated next to their route handlers

## Deployment

Standard procedure (identical across all six company repos):

1. Merge to main
2. Wait for CI to pass
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. If p95 latency exceeds 500ms, page on-call
7. `./deploy prod`
8. Announce in #releases
