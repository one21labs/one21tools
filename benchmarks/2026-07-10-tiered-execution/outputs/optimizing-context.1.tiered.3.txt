## Project Overview

## Critical Rules

### Money: Integers Only

**All amounts are integer cents. Floats for money are banned.**

In 2024, a float-rounding incident corrupted invoices. Never use floating-point arithmetic for monetary calculations. Amounts must be stored, passed, and computed as integers representing cents.

---

## Testing

Two separate test commands with different requirements:

- **`npm test`** — Playwright E2E suite. **Requires a running dev server** or it will fail.
- **`npm run test:unit`** — Unit tests.

Always start the dev server before running `npm test`.

---

## Database Migrations

**Always use `./scripts/migrate.sh` for migrations.**

Do not run `prisma migrate` directly. Running it directly breaks the shadow database that CI depends on.

---

## Authentication

API routes authenticate with a custom `X-Session-Token` header scheme, not standard Bearer tokens.

Refer to `docs/api.md` for the full authentication scheme documentation.

---

## Code Style

Follow the 45-page style guide at `docs/style-guide.md`.

Deviations from typical TypeScript style:
- No default exports
- Zod schemas are colocated next to their route handlers

## Deployment

Deployment procedure (identical across all six company repos):

1. Merge to main
2. Wait for CI to go green
3. Run `npm run build:prod`
4. Run `./deploy staging`
5. Smoke-test `/healthz` on staging
6. If p95 latency exceeds 500ms, page on-call; otherwise continue
7. Run `./deploy prod`
8. Announce the release in `#releases`

Do not skip steps or run them out of order.
