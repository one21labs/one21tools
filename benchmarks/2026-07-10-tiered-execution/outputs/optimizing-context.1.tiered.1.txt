# checkout-service

**Stack:** Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

**Owned by:** Payments squad. (Originally a Rails prototype from 2021; rewritten in 2023 by the platform team.)

## Getting Started

See README.md for full onboarding (clone, npm install, .env setup, npm run dev).

## Critical Rules

### Money Handling
- **All amounts are integer cents. Floats for money are banned.**
- This rule exists because a 2024 float-rounding incident corrupted invoices.

### Database Migrations
- **Always use `./scripts/migrate.sh` — never run `prisma migrate` directly.**
- Running `prisma migrate` directly breaks the shadow database that CI relies on.

## Testing

- **`npm test`**: Runs the Playwright E2E suite. **Requires a running dev server or it will fail.** Start it with `npm run dev` first.
- **`npm run test:unit`**: Runs unit tests.

## API Authentication

API routes authenticate using a custom **`X-Session-Token`** header. This is **not** a standard Bearer token scheme.

See `docs/api.md` for the full authentication scheme documentation.

## Style Guide

See `docs/style-guide.md` for the complete style guide (45 pages).

Deviations from typical TypeScript style:
- No default exports (use named exports only).
- Zod schemas are colocated next to their route handlers.

## Deployment

Deployment procedure (identical across all six company repos):

1. Merge to main.
2. Wait for CI to turn green.
3. Run `npm run build:prod`.
4. Run `./deploy staging`.
5. Smoke-test `/healthz` endpoint on staging.
6. If p95 latency exceeds 500ms, page the on-call engineer.
7. Run `./deploy prod`.
8. Announce the release in #releases.

Do not skip or reorder these steps.
