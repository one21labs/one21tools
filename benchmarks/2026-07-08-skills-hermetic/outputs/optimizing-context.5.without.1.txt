```markdown
# Project Instructions for Claude

## Code Style
- Prefer `const` over `let`.
- Use meaningful, descriptive names.
- Components: PascalCase. Hooks: prefixed with `use`.
- Keep functions small.
- Avoid deeply nested ternaries.

## API Authentication — IMPORTANT, custom scheme (not Bearer)
Requests to `/api/*` must carry an `X-Client-Sig` header: an HMAC-SHA256
signature of the request body, generated using the key from
`scripts/sign.mjs`. Full details in `docs/auth.md`.

Do NOT use `Authorization: Bearer ...` for `/api/*` — that is wrong for
this project and will fail. Always sign with `X-Client-Sig` instead.

## Testing
- `pnpm test` — full suite; launches real Chromium instances and requires
  the dev server to be running. Slow, not for iteration.
- `pnpm test:unit` — use this for fast local iteration.

## Onboarding / Local Setup
(mirrors README.md — keep both in sync if this changes)
1. Clone the repo
2. `pnpm install`
3. Copy `.env.example` to `.env`
4. `pnpm dev`
5. Open http://localhost:3000

## Deployment
Deploys follow a strict, ordered 14-step procedure — see `runbook.md`.
Do not run `./ship` or any deploy/release command unless the user
explicitly asks to deploy. Follow the steps in order; don't skip the
healthcheck, smoke suite, or canary wait.
```

**`runbook.md`**

```markdown
# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`.
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`.
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`.
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`.
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`.
13. Verify Sentry release tagging.
14. Announce in #releases.
```
