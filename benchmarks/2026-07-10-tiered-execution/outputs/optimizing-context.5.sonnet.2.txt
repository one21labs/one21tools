# Project Instructions for Claude

## Code Style
- Prefer `const` over `let`.
- Use meaningful, descriptive names.
- Components: PascalCase. Hooks: prefixed with `use`.
- Keep functions small.
- Avoid deeply nested ternaries.

## API Auth (IMPORTANT — do not use Bearer auth)
Requests to `/api/*` do NOT use standard Bearer token auth. They require a custom
`X-Client-Sig` header: an HMAC-SHA256 signature of the request body, generated with
the key from `scripts/sign.mjs`. Full details in `docs/auth.md`.

If you catch yourself writing `Authorization: Bearer ...` for an `/api/*` request,
stop — that is wrong for this project. Use the signing flow in `scripts/sign.mjs` instead.

## Tests
- `pnpm test` launches real Chromium instances and requires the dev server running —
  slow, don't use it for quick iteration.
- `pnpm test:unit` for fast iteration.

## Onboarding (mirrors README.md)
1. Clone the repo.
2. `pnpm install`
3. Copy `.env.example` to `.env`.
4. `pnpm dev`
5. Open `localhost:3000`.

## Deploying
Deploy steps are NOT summarized here — follow `runbook.md` exactly, in order, including
the abort/rollback conditions. Do not skip steps or improvise deploy commands.
```

# Deploy Runbook

Follow in order. Do not skip steps.

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if error rate > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```
