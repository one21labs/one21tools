## Decision
1. **`retrospect`: opus -> sonnet** (one frontmatter line, `agents/retrospect.md:4`). It is the single agent off the gating path: post-ship, non-gating, output is advice the orchestrator independently re-verifies (`retrospect.md:42-44`). Dropping it makes the strong-tier set exactly {pm, red-team, verifier} = the doctrine's lead/gate/adversary, so this ALIGNS the doctrine rather than excepting it.
2. **Keep** pm=opus, verifier=opus, red-team=opus (lead/gate/adversary — doctrine-pinned, each fails silent+uncaught+load-bearing, a missed BLOCKER ships to every consumer), tech-lead=sonnet (bridge — inherited, out of scope).
3. **Extend the doctrine line** at `decide/SKILL.md:53` (advisor C's poka-yoke, prevent>detect) to classify the Act-loop role->tier so the frontmatter is derived-and-explained, not a tier any reader can "tidy" in either direction. EXACT final change — replace the parenthetical `(lead / gate / adversary on the strongest model, advisors + bridge cheaper)` with `(lead / gate / adversary on the strongest model, advisors + bridge + the Act-loop retrospective cheaper)`. The first half already classifies the strong-tier trio (literal roles per `decide/SKILL.md:42-46`: lead=pm, gate=verifier, adversary=red-team); the edit ONLY appends the Act-loop retrospective to the cheaper clause — no trailing gloss. Deliberately NO "only the gating roles need the strong tier" wording: verifier is the only literal gate (`:44`), so "gating roles" would contradict the same sentence's first half and license dropping pm + red-team to sonnet (the cut this ADR forbids on the two silent+uncaught+load-bearing agents). One line, no per-agent prose (that would be muda); WHY lives in this ADR, the role->tier classification in the doctrine line.

## Justification
retrospect has two failure modes, and BOTH are backstopped — neither blocks a ship or writes a user-facing record. (a) Fabrication (pad a commit / thin findings) is loud + checkable (cite-existence tripwire, `retrospect/SKILL.md:33`). (b) Wrong-home routing with REAL cites (e.g. filing a recurrence-prevention fix as a prose rule when its lowest home is a test/hook or a CLAUDE.md inviolable) is the quieter, capability-sensitive mode — the agent owns the routing rules (`SKILL.md:30`) and the cite-tripwire does NOT catch it. Its backstop is not "loudness" but the orchestrator's muda-assess-before-apply on the cheap non-judgment routings PLUS the escalation rule: any judgment-call routing goes to `/decide`+ADR (`retrospect.md:42-44`), so the high-stakes routings are PM-verified before they land. This (b)-mode is exactly what the weakest [checkable] A/B tests, so the cut's soundness rides on that test, not on an "always loud" claim. Effort trivial (1 line), risk low (non-gating + re-checked), value modest-but-compounding (every consumer inherits; `/retrospect` runs before every PR per CLAUDE.md). The doctrine-line extension is the strongest reason to act beyond the bare cut: retrospect's tier is the only one the SSoT line leaves unclassified, so without it the tier is a drive-by-reversible outlier (detect-not-prevent); classifying the role->tier in the line itself is the poka-yoke. Cost x risk x value favors both; neither touches a gating path.
