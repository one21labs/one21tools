# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Stack
Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

## History & Ownership
Originally a Rails prototype (2021), rewritten by the platform team in 2023. Currently owned by the payments squad.

## Money Handling
**All monetary amounts are integer cents. Never use floats for money.**
A float-rounding bug in 2024 corrupted invoices — this is a hard rule, not a style preference. When adding fields, doing arithmetic, or parsing money values, keep everything in integer cents until final display formatting.

## Testing
- `npm test` runs the Playwright E2E suite — requires a running dev server, or it fails. Start `npm run dev` first (or in another terminal) before running it.
- `npm run test:unit` runs unit tests and does not need a dev server.

## Database Migrations
Always use `./scripts/migrate.sh` for migrations. Do **not** run `prisma migrate` directly — it breaks the shadow database that CI depends on.

## Authentication
API routes authenticate via a custom `X-Session-Token` header — this is not standard Bearer token auth. Full scheme documented in `docs/api.md`; consult it before implementing or modifying auth-related code.

## Code Style
Full style guide at `docs/style-guide.md` (45 pages). Two deviations from typical TypeScript conventions to remember:
- No default exports.
- Zod schemas are colocated next to their route handlers (not in a separate schemas directory).

## Deployment
Same procedure used across all six company repos:
1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`

## Onboarding
See README.md for setup (clone, install, env, dev server).
