# Project Guidelines

## About This Project
Internal orders dashboard, built with Flask.

## Setup
See README.md.

## Code Style
- Python 3.11, PEP 8, 4-space indentation
- f-strings over `.format()`
- Docstrings on public functions

## Database
- Orders table schema is at version 12 (`models.py`)
- Never use raw SQL

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere — naive datetimes caused a 2023 bug that double-charged customers in CET
- Run `make lint` before committing; CI runs the same target
- Local tests: `pytest -m "not integration"`; integration tests need the docker compose stack up
