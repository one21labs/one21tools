name: json-config-check
description: Use when a JSON config file needs to be validated for syntax errors or checked against a JSON Schema before use/deployment. Parses the file, validates it, and reports errors with location and reason.
---

# JSON Config Check

## Validate syntax

```python
import json

with open("config.json") as f:
    try:
        data = json.load(f)
    except json.JSONDecodeError as e:
        print(f"Invalid JSON: {e.msg} at line {e.lineno}, column {e.colno}")
        raise
```

## Validate against a schema

If a schema file exists (e.g. `config.schema.json` next to the config, or a
`$schema` key inside it), validate with the `jsonschema` package:

```python
from jsonschema import validate, ValidationError

with open("config.schema.json") as f:
    schema = json.load(f)

try:
    validate(instance=data, schema=schema)
except ValidationError as e:
    path = ".".join(str(p) for p in e.path) or "<root>"
    print(f"Schema violation at {path}: {e.message}")
    raise
```

If no schema file is found, skip schema validation and report syntax-only
results — don't fabricate a schema.

## Reporting

For each error, report: file path, location (line/column for syntax errors,
JSON path for schema errors), and the specific message. Treat the file as
invalid until every reported error is resolved.
```
