# CLAUDE.md

## Stack
Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

## Money handling
All monetary amounts are integer cents. **Never use floats for money** — a 2024 float-rounding
incident corrupted invoices. Any new code touching prices, totals, or balances must use integer
cents (or an existing Decimal/BigInt helper if one exists in the codebase — check before adding
a new one).

## Tests
- `npm test` runs the Playwright E2E suite — it requires a running dev server and will fail without one. Don't run it as a quick sanity check; start `npm run dev` first or use it deliberately.
- `npm run test:unit` runs unit tests — this is the fast, no-server-needed option for iterating.

## Database migrations
Always create/apply migrations via `./scripts/migrate.sh`. Do **not** run `prisma migrate`
directly — it breaks the shadow database CI relies on.

## Auth
API routes authenticate via a custom `X-Session-Token` header scheme — this is **not** standard
Bearer/OAuth. Full scheme details are in `docs/api.md`; read that before touching auth-related
route handlers.

## Style
Full style guide: `docs/style-guide.md`. Otherwise-standard TypeScript style, with two
project-specific deviations:
- No default exports.
- Zod schemas are colocated next to the route handler they validate, not in a shared schemas file.

## Deploys
Same procedure across all six company repos — merge to `main` → CI green → `npm run build:prod`
→ `./deploy staging` → smoke-test `/healthz` → page on-call if p95 latency > 500ms →
`./deploy prod` → announce in `#releases`.

This is a manual, gated process with production and paging side effects — don't run deploy
steps autonomously. Treat each step as requiring explicit confirmation, even if a prior step
was approved.

## Onboarding
See `README.md` (clone, `npm install`, copy `.env.example` to `.env`, `npm run dev`).
