```markdown
# CLAUDE.md

Guidance for Claude Code when working in checkout-service.

## Project context

Owned by the payments squad. Originally a Rails prototype (2021), rewritten
in Next.js by the platform team in 2023.

- **Stack**: Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres.
- **Package manager**: npm — use standard `npm install` / `npm run` workflows.

## Money handling

All monetary amounts are integer cents. **Never use floats for money** —
a float-rounding bug in 2024 corrupted invoices. Watch for this in new code,
migrations, and third-party API integrations.

## Testing

- `npm test` — Playwright E2E suite. **Requires a running dev server**; it
  will fail if one isn't up. Don't treat that failure as a code bug without
  checking first.
- `npm run test:unit` — unit tests, no server required.

## Database migrations

Always run migrations through `./scripts/migrate.sh`. Never call
`prisma migrate` directly — it breaks the shadow database CI depends on.

## Auth

API routes authenticate via a custom `X-Session-Token` header, not standard
Bearer tokens. Full scheme documented in `docs/api.md` — check it before
writing or reviewing auth-related code.

## Code style

Full style guide: `docs/style-guide.md`. Two notable deviations from typical
TypeScript conventions:

- No default exports.
- Zod schemas live colocated next to their route handlers, not in a shared
  schemas directory.

## Deploy procedure

Same process used across all six company repos:

1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. If p95 latency exceeds 500ms, page on-call.
7. `./deploy prod`.
8. Announce in `#releases`.

Do not run deploy steps unattended — confirm with the user before executing
`./deploy staging` / `./deploy prod`, since these affect shared/production
systems.

## Onboarding

See `README.md` (clone → `npm install` → copy `.env.example` to `.env` →
`npm run dev`).
```
