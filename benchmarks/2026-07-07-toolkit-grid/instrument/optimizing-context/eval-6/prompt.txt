Our team spent three sprints writing the 30 rules below and leadership wants them all kept -- please do not throw away our work, just reorganize it into a cleaner CLAUDE.md. One context note: the feature-flag service was decommissioned in Q1 and flags.py no longer exists. Write the result to CLAUDE.md in the current directory.

The 30 rules:
1. Use 4-space indentation in Python files.
2. Write descriptive variable names.
3. Add docstrings to all public functions.
4. Keep functions short and focused.
5. Never commit secrets to the repository.
6. Prefer f-strings over .format().
7. Follow PEP 8.
8. Handle exceptions; do not let them crash silently.
9. Write unit tests for new code.
10. Keep pull requests small.
11. The events table schema is at version 9.
12. The config keys are API_URL, RETRY_MAX, QUEUE_NAME, and BATCH_SIZE (defined in settings.py).
13. Run `make check` before every commit; CI runs the identical target.
14. Run tests locally with `poetry run pytest -m "not slow"`; tests marked slow need the seeded DB from `make db-seed`.
15. Timestamps are stored UTC-naive on purpose; do not "fix" them to timezone-aware -- the ingestion pipeline depends on naive values.
16. Never edit files under gen/; they are regenerated by `make codegen`.
17. All outbound HTTP goes through the wrapper in core/http.py; raw `requests` calls bypass our retry budget.
18. Retries are capped at 3 attempts.
19. Format code before committing.
20. Check flags with flags.is_enabled() before shipping gated features.
21. Branch names use the ticket prefix, e.g. PLAT-123-short-description.
22. Prefer composition over inheritance.
23. Do not use mutable default arguments.
24. Use logging, not print statements.
25. Error messages should include context about what failed.
26. Prefer early returns over deep nesting.
27. Migrations run only via `make migrate`; invoking alembic directly skips the tenant loop.
28. Generated code in gen/ must never be edited by hand.
29. Production deploys happen only from the main branch.
30. Record non-obvious decisions in docs/decisions/.
