```markdown
# Project Instructions

## Style Rules
- Prefer `const` over `let`.
- Use meaningful, descriptive names.
- Components in PascalCase; hooks prefixed with `use`.
- Keep functions small.
- Avoid deeply nested ternaries.

## API Authentication
Requests to `/api/*` must carry an `X-Client-Sig` header: an HMAC-SHA256 signature
of the request body, generated using the key from `scripts/sign.mjs`.

This is **custom signature auth, NOT Bearer token auth**. Do not write standard
`Authorization: Bearer ...` code for these endpoints — it will fail. Full details
in `docs/auth.md`.

## Tests
- `pnpm test` launches real Chromium instances and requires the dev server
  running. Slow — use for full validation, not iteration.
- `pnpm test:unit` — fast, use this while iterating.

## Onboarding
1. Clone the repo.
2. `pnpm install`
3. Copy `.env.example` to `.env`.
4. `pnpm dev`
5. Open `localhost:3000`.

## Deploy
Full production deploy procedure lives in `runbook.md` — follow it step by step,
in order, no skipping steps.
```

```markdown
# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```
