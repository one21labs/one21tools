Material to merge:

[Style rules] Prefer const over let. Use meaningful, descriptive names. Components in PascalCase, hooks prefixed with use. Keep functions small. Avoid deeply nested ternaries.

[API auth] Requests to /api/* must carry an X-Client-Sig header: an HMAC-SHA256 of the request body using the key from scripts/sign.mjs. This is custom, not Bearer auth; details in docs/auth.md. Claude keeps writing standard Bearer code here and it always fails.

[Tests] `pnpm test` launches real Chromium instances and needs the dev server; for iteration use `pnpm test:unit`.

[Onboarding, same text as README.md] Clone the repo, run `pnpm install`, copy `.env.example` to `.env`, run `pnpm dev`, open localhost:3000.

[Deploy runbook, 14 steps] 1. Confirm main is green. 2. `pnpm build`. 3. `pnpm run bundle-analyze`, abort if main chunk > 900kB. 4. Tag release vX.Y.Z. 5. `./ship staging`. 6. Wait for staging healthcheck. 7. Run smoke suite `pnpm smoke:staging`. 8. Verify checkout flow manually with test card 4242. 9. Post staging sign-off in #launch. 10. `./ship prod --canary 5`. 11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%. 12. `./ship prod --full`. 13. Verify Sentry release tagging. 14. Announce in #releases.
