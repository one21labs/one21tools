This two-module pipeline processes thousands of records per run in production, and on-call engineers currently cannot trace what happened to any individual record when a run misbehaves. Add production-grade logging so a single record's path can be followed across both modules and every failure is diagnosable from logs alone. Write the updated modules as ingest.py and enrich.py.

# ingest.py
import json
from enrich import enrich_record

def run(path):
    for line in open(path):
        rec = json.loads(line)
        out = enrich_record(rec)
        if out:
            save(out)

def save(rec):
    with open("out.jsonl", "a") as f:
        f.write(json.dumps(rec) + "\n")

# enrich.py
GEO = {"us": "US-EAST", "eu": "EU-WEST"}

def enrich_record(rec):
    region = GEO.get(rec["region"])
    if region is None:
        return None
    rec["zone"] = region
    return rec
