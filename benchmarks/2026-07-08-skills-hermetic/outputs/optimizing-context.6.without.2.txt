# CLAUDE.md

## Code Style
- Use 4-space indentation in Python files. (#1)
- Follow PEP 8. (#7)
- Write descriptive variable names. (#2)
- Add docstrings to all public functions. (#3)
- Keep functions short and focused. (#4)
- Prefer f-strings over `.format()`. (#6)
- Prefer composition over inheritance. (#22)
- Do not use mutable default arguments. (#23)
- Prefer early returns over deep nesting. (#26)
- Use logging, not print statements. (#24)
- Error messages should include context about what failed. (#25)
- Handle exceptions; do not let them crash silently. (#8)

## Testing
- Write unit tests for new code. (#9)
- Run tests locally with `poetry run pytest -m "not slow"`; tests marked
  slow need the seeded DB from `make db-seed`. (#14)

## Git & CI Workflow
- Never commit secrets to the repository. (#5)
- Keep pull requests small. (#10)
- Branch names use the ticket prefix, e.g. `PLAT-123-short-description`. (#21)
- Format code before committing. (#19)
- Run `make check` before every commit; CI runs the identical target. (#13)
- Production deploys happen only from the main branch. (#29)

## Guardrails — Do Not Bypass
- Never edit files under `gen/`; they are regenerated by `make codegen`. (#16, #28)
- All outbound HTTP goes through the wrapper in `core/http.py`; raw
  `requests` calls bypass our retry budget. (#17)
- Retries are capped at 3 attempts. (#18)
- Migrations run only via `make migrate`; invoking alembic directly
  skips the tenant loop. (#27)
- Timestamps are stored UTC-naive on purpose; do not "fix" them to
  timezone-aware — the ingestion pipeline depends on naive values. (#15)
- **⚠️ Outdated — needs replacement:** Previously, gated features were
  checked via `flags.is_enabled()` in `flags.py`. That feature-flag
  service was decommissioned in Q1 and `flags.py` no longer exists.
  This rule is kept for traceability but should not be followed as-is;
  replace with whatever gating mechanism (if any) is current before
  relying on it. (#20)

## Configuration Reference
- The events table schema is at version 9. (#11)
- Config keys are `API_URL`, `RETRY_MAX`, `QUEUE_NAME`, and
  `BATCH_SIZE` (defined in `settings.py`). (#12)

## Documentation
- Record non-obvious decisions in `docs/decisions/`. (#30)
