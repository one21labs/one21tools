```markdown
# CLAUDE.md

## Project context
checkout-service is owned by the payments squad. It started as a 2021 Rails prototype and was
rewritten in 2023 by the platform team — don't expect Rails idioms or artifacts to remain.

Stack: Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

## Money handling — hard rule
All monetary amounts are **integer cents**. Never use floats for money.
A float-rounding bug in 2024 corrupted invoices in production — treat this as non-negotiable,
not a style preference. Reject/flag any code that stores or computes money as a float/decimal.

## Testing
- `npm test` — runs the Playwright E2E suite. Requires a running dev server; it will fail if one
  isn't up. Don't treat that failure as a code regression without checking first.
- `npm run test:unit` — unit tests, no dev server needed.

## Database migrations
Always create/apply migrations via `./scripts/migrate.sh`. Never run `prisma migrate` directly —
it breaks the shadow database CI relies on.

## Auth
API routes authenticate via a custom `X-Session-Token` header scheme — this is **not** standard
Bearer/OAuth. Full scheme is documented in `docs/api.md`; read it before touching auth code.

## Style
Full style guide: `docs/style-guide.md`. Otherwise-standard TypeScript style, with two deviations:
- No default exports.
- Zod schemas are colocated next to the route handlers that use them.

## Deploys
Same procedure as our other repos:
1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`

## Onboarding
See README.md (clone → `npm install` → copy `.env.example` to `.env` → `npm run dev`).
```
