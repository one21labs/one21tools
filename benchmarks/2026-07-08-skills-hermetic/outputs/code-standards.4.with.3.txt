```python
def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, TypeError):
            continue
    return prices
```

What changed: each line is now parsed inside a `try/except`, so a malformed line (wrong number of fields, non-numeric price) is skipped instead of raising and aborting the whole batch. Everything else is untouched, as requested.
