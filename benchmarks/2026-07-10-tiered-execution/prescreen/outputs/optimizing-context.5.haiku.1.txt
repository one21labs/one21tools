```markdown
# Project Guidelines & Runbook

## Quick Start (Onboarding)

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open http://localhost:3000

## Code Style

- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase, hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Authentication

**Critical:** All requests to `/api/*` must include an `X-Client-Sig` header.

This is **not** Bearer auth. The header value is an HMAC-SHA256 of the request body using the key from `scripts/sign.mjs`.

See `docs/auth.md` for full details. Claude frequently writes standard Bearer auth here and it always fails—do not do this.

## Testing

- `pnpm test` — launches real Chromium instances; requires the dev server running
- `pnpm test:unit` — for fast iteration without starting Chromium

## Deployment Runbook

**Friday Launch:** Follow these 14 steps exactly.

1. **Confirm main is green** — all CI checks pass
2. **Build:** `pnpm build`
3. **Bundle check:** `pnpm run bundle-analyze` — abort if main chunk > 900kB
4. **Tag release:** `git tag vX.Y.Z` (e.g., v1.2.3)
5. **Ship to staging:** `./ship staging`
6. **Wait:** Staging healthcheck must pass
7. **Smoke tests:** `pnpm smoke:staging`
8. **Manual checkout:** Test the full flow with card `4242`
9. **Sign-off:** Post staging sign-off in #launch
10. **Canary deploy:** `./ship prod --canary 5`
11. **Monitor 15 minutes:** Watch error rate in Sentry; if > 0.5%, rollback via `./ship rollback`
12. **Full deploy:** `./ship prod --full`
13. **Release tagging:** Verify Sentry release is tagged correctly
14. **Announce:** Post in #releases

Do not skip steps or parallelize. Each step depends on the previous one passing.
```
