# JSON Config Check

## Purpose

Check a JSON configuration file in two stages: (1) confirm it is syntactically
valid JSON, (2) if a JSON Schema for it can be found, validate the config
against that schema.

## Instructions

1. **Identify the target file** — the config path given by the user, or
   inferred from context (e.g. `config.json`, `settings.json`).

2. **Check syntax first.** Run one of:
   - Python: `python -c "import json; json.load(open('PATH'))"`
   - Node: `node -e "JSON.parse(require('fs').readFileSync('PATH', 'utf8'))"`

   If this fails, report the parser's exact error message (it includes
   line/column) to the user and **stop** — do not attempt schema validation
   on invalid JSON.

3. **Look for a schema**, in this order, and use the first match:
   - A `"$schema"` key inside the config file pointing to a local path or URL.
   - A sibling file named `schema.json` or `<config-basename>.schema.json`.

   If none of these exist, report that the file is syntactically valid JSON
   and stop. Do not invent or assume a schema.

4. **Validate against the schema**, if one was found. Confirm `jsonschema` is
   installed first (`python -c "import jsonschema"`); if it isn't, tell the
   user to run `pip install jsonschema` before continuing. Then run:

   ```python
   import json
   from jsonschema import validate, ValidationError

   config = json.load(open("config.json"))
   schema = json.load(open("schema.json"))
   try:
       validate(instance=config, schema=schema)
   except ValidationError as e:
       print(f"Schema validation failed at {list(e.path)}: {e.message}")
   ```

5. **Report results**, always including concrete detail rather than a bare
   pass/fail:
   - Syntax error: file path, line/column, and the parser's message verbatim.
   - Schema error: the failing JSON path, the violated constraint, and the
     actual value.
   - Success: state explicitly whether the check covered syntax only (no
     schema found) or syntax + schema.

## Notes

- Never suppress parser/validator output — surface it verbatim so the user
  can act on it directly.
- Don't guess at a schema's existence or contents; only validate against a
  schema you actually located per step 3.
```
