```markdown
# CLAUDE.md — checkout-service

Payments-squad service handling checkout. Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

Setup (clone, install, env, dev server) is in **README.md** — start there.

## Hard rules

- **Money is always integer cents. Never use floats for monetary amounts.** A 2024 float-rounding bug corrupted invoices; this is banned, not a preference.
- **Run migrations only through `./scripts/migrate.sh`.** Calling `prisma migrate` directly breaks the shadow database CI depends on.
- **No default exports.** Use named exports.
- **Colocate Zod schemas** in the same file as the route handler they validate.

## Tests

- `npm test` runs the Playwright E2E suite. It requires a running dev server and will fail without one.
- `npm run test:unit` runs unit tests (no server needed).

## Auth

API routes use a custom `X-Session-Token` header scheme — **not** standard Bearer tokens. Full scheme is in **docs/api.md**; consult it before touching auth.

## Style

Full 45-page guide: **docs/style-guide.md**. It's standard TypeScript style except for the two deviations noted above (no default exports; colocated Zod schemas).

## Deploys

Standard company procedure (identical across all six repos):
1. Merge to main
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in `#releases`
```
