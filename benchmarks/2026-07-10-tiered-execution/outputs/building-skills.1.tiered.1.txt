# Sanitize Customer Exports

## Purpose

This skill removes personally identifiable information (PII) from customer CSV files to ensure compliance before external sharing. It applies fixed redaction rules consistently across all exports.

## When to Use

Invoke this skill when:
- A user requests to redact or sanitize a customer CSV file before export
- A CSV containing customer data needs PII removal (email, phone, SSN, etc.)
- A file is about to be shared externally and must have sensitive data masked

## How It Works

The skill applies these **fixed redaction rules**:

1. **Dropped columns** (removed entirely): `email`, `phone`, `ssn`
2. **Name column**: Replaced with an opaque sequential ID (e.g., `ID_0001`, `ID_0002`)
3. **Free-text columns** (`notes`, `comments`): Email addresses and phone numbers within the text are masked with `[EMAIL_REDACTED]` and `[PHONE_REDACTED]`
4. **All other columns**: Preserved as-is

## Usage

To redact a CSV file:

```bash
python scripts/redact_csv.py <path/to/input.csv>
```

**Input:** Any CSV file (e.g., `customers.csv`)  
**Output:** `<input>.redacted.csv` (e.g., `customers.csv.redacted.csv`)  
**Important:** The original file is never modified in place.  

### Output Filename Convention

The output filename appends `.redacted.csv` literally to the input filename. For example:
- Input: `customers.csv` → Output: `customers.csv.redacted.csv`
- Input: `data.txt` → Output: `data.txt.redacted.csv`

This convention allows multiple redacted versions to coexist with their originals without filename collisions.

## Output Format

- The redacted CSV is written with UTF-8 encoding
- Column order is preserved (except dropped columns)
- All data rows are processed consistently
- All columns except `email`, `phone`, `ssn` are retained (transformed if applicable)

## Example

**Input:** `customers.csv`
```
name,email,phone,ssn,notes,comments,address
Alice Smith,alice@example.com,555-1234,123-45-6789,"Follow up: contact alice@example.com","Support ticket #001 for 555-1234",123 Main St
Bob Jones,bob@example.com,555-5678,234-56-7890,"No notes","Customer since 2020",456 Oak Ave
Carol White,carol.white@company.com,555.345.6789,345-67-8901,"Email: carol.white@company.com","Referred by alice@example.com",789 Pine Rd
```

**Output:** `customers.csv.redacted.csv`
```
name,notes,comments,address
ID_0001,"Follow up: contact [EMAIL_REDACTED]","Support ticket #001 for [PHONE_REDACTED]",123 Main St
ID_0002,"No notes","Customer since 2020",456 Oak Ave
ID_0003,"Email: [EMAIL_REDACTED]","Referred by [EMAIL_REDACTED]",789 Pine Rd
```

## Technical Details

- **Email detection**: Standard email format patterns with domain validation
- **Phone detection**: North American phone numbers in common formats (10-digit, with dashes/parentheses/dots)
- **Name ID generation**: Sequential per-file (ID_0001, ID_0002, ...), deterministic and non-reversible
- **Column matching**: Case-insensitive (`email`, `Email`, `EMAIL` all matched)

## Requirements

- Python 3.6+
- No external dependencies (uses only standard library)

## Error Handling

- Non-existent input file: Reports error and exits with status 1
- Malformed CSV: Processes best-effort; reports errors to stderr
- Empty CSV: Reports error and exits

### **File 2: `outputs/sanitizing-customer-exports/scripts/redact_csv.py`**

```python
#!/usr/bin/env python3

import csv
import re
import sys
import os


EMAIL_PATTERN = r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b'
PHONE_PATTERN = r'\b(\+?1[-.\s]?)?\(?[0-9]{3}\)?[-.\s]?[0-9]{3}[-.\s]?[0-9]{4}\b'


def mask_pii_in_text(text):
    text = re.sub(EMAIL_PATTERN, '[EMAIL_REDACTED]', text)
    text = re.sub(PHONE_PATTERN, '[PHONE_REDACTED]', text)
    return text


def redact_csv(input_file):
    if not os.path.isfile(input_file):
        print(f"Error: File not found: {input_file}", file=sys.stderr)
        return False
    
    output_file = f"{input_file}.redacted.csv"
    
    try:
        with open(input_file, 'r', encoding='utf-8') as infile:
            reader = csv.DictReader(infile)
            
            if reader.fieldnames is None:
                print(f"Error: {input_file} is empty or not a valid CSV", file=sys.stderr)
                return False
            
            columns_to_drop = {'email', 'phone', 'ssn'}
            free_text_columns = {'notes', 'comments'}
            
            fieldnames_lower = {fn.lower(): fn for fn in reader.fieldnames}
            
            cols_to_drop_actual = {fieldnames_lower[col] for col in columns_to_drop if col in fieldnames_lower}
            free_text_cols_actual = {fieldnames_lower[col] for col in free_text_columns if col in fieldnames_lower}
            
            output_fieldnames = [fn for fn in reader.fieldnames if fn not in cols_to_drop_actual]
            
            with open(output_file, 'w', encoding='utf-8', newline='') as outfile:
                writer = csv.DictWriter(outfile, fieldnames=output_fieldnames)
                writer.writeheader()
                
                row_index = 0
                for row in reader:
                    row_index += 1
                    output_row = {}
                    
                    for field in output_fieldnames:
                        value = row.get(field, '')
                        
                        if field.lower() == 'name':
                            value = f"ID_{row_index:04d}"
                        elif field in free_text_cols_actual:
                            value = mask_pii_in_text(value)
                        
                        output_row[field] = value
                    
                    writer.writerow(output_row)
        
        print(f"Redacted CSV written to: {output_file}")
        return True
        
    except Exception as e:
        print(f"Error processing CSV: {e}", file=sys.stderr)
        return False


if __name__ == '__main__':
    if len(sys.argv) != 2:
        print("Usage: redact_csv.py <input_csv_file>", file=sys.stderr)
        sys.exit(1)
    
    success = redact_csv(sys.argv[1])
    sys.exit(0 if success else 1)
```
