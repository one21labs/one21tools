```markdown
# Project Guide

## Getting Started
- Clone the repo
- `pnpm install`
- Copy `.env.example` to `.env`
- `pnpm dev`
- Open http://localhost:3000

## Code Style
- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components: PascalCase
- Hooks: prefix with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Authentication — IMPORTANT, not Bearer auth
Requests to `/api/*` do NOT use Bearer tokens. They require a custom
`X-Client-Sig` header: an HMAC-SHA256 signature of the request body,
generated using the key in `scripts/sign.mjs`. Full details: `docs/auth.md`.

Do not write `Authorization: Bearer ...` for these routes — it will fail.
Always sign the body and send `X-Client-Sig` instead.

## Testing
- `pnpm test` — full e2e suite. Launches real Chromium instances and
  requires the dev server to be running. Slow — not for iteration.
- `pnpm test:unit` — fast, use this while iterating.

## Deploying
Full procedure lives in `runbook.md`. Do not run any deploy steps unless
explicitly asked to deploy.
```

**runbook.md**
```markdown
# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```
