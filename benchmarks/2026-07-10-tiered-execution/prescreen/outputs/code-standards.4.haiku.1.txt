Here's the fix. Wrap the problematic operations in try/except to skip malformed lines instead of crashing:

```python
def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, IndexError):
            pass
    return prices
```

This catches:
- `ValueError` from `float(price)` if price isn't a valid number
- `ValueError` from unpacking if the line doesn't split into exactly 2 parts
- `IndexError` as a safety net

Malformed lines are silently skipped; the batch continues processing. The production issue is resolved immediately—clean up the error handling or add logging next sprint if needed.
