```markdown
# CLAUDE.md — checkout-service

Payment checkout service owned by the **payments squad**. Next.js 14 (App Router),
TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.

New to the repo? Onboarding (clone, install, env, dev server) is in **README.md** — start there.

## Critical rules

These have bitten us before. Do not violate them.

- **Money is always integer cents.** Never use floats or decimals for monetary amounts.
  A 2024 float-rounding bug corrupted invoices; floats for money are banned.
- **Run migrations only through `./scripts/migrate.sh`.** Calling `prisma migrate`
  directly breaks the shadow database that CI relies on.

## Testing

- `npm run test:unit` — unit tests. Use this for quick feedback.
- `npm test` — Playwright E2E suite. **Requires a running dev server** (`npm run dev`)
  and fails without one.

## Auth

API routes authenticate with a custom **`X-Session-Token`** header — *not* standard
Bearer tokens. The full scheme is documented in **docs/api.md**; read it before
touching auth or adding an authenticated route.

## Code style

Full style guide is **docs/style-guide.md** (the source of truth). It's standard
TypeScript style with two deviations to remember:

- **No default exports** — use named exports everywhere.
- **Colocate Zod schemas** next to their route handlers, not in a shared schemas file.

## Deploy

Standard company deploy procedure (identical across all six repos):

1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. If p95 latency exceeds 500ms, page on-call (do not continue).
7. `./deploy prod`.
8. Announce in **#releases**.
```
