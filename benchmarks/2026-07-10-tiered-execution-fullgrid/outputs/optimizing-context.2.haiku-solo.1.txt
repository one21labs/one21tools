```markdown
# Project Guidelines

## About This Project
This is a Python web application built with Flask. We chose it for its simplicity and flexibility. The app serves an internal orders dashboard.

## Code Style
We write Python 3.11. Always use 4 spaces for indentation as per PEP 8. Use meaningful variable names, docstrings for public functions, short focused functions, and f-strings over `.format()`.

## Database
The orders table schema is currently at version 12 (defined in models.py). Never use raw SQL.

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere; a 2023 bug from naive datetimes double-charged customers in CET.
- Run `make lint` before committing; CI runs the exact same target.
- Local tests: `pytest -m "not integration"`; the integration marker needs the docker compose stack up.
```
