# JSON Config Check

Validates a JSON configuration file in two stages: syntax, then (if a
schema is available) schema compliance.

## Procedure

1. **Locate the target file.** Use the path the user gave, or if they named
   a directory, find `*.json` files in it (non-recursive unless asked).

2. **Check syntax.** Parse the file with `python3 -c "import json,sys;
   json.load(open(sys.argv[1]))" <file>` (or the language's standard JSON
   parser). If parsing fails, report the exact error message and line/column
   from the parser and stop — do not proceed to schema validation.

3. **Look for a schema.** In the same directory as the config file, look
   for a schema file matching one of these, in order:
   - a `$schema` key inside the config file itself (a path or URL)
   - `<config-basename>.schema.json`
   - `schema.json`

   If none is found, report that syntax passed and that no schema was
   available for further checks, then stop.

4. **Validate against the schema.** Use Python's `jsonschema` package:
   ```python
   import json
   from jsonschema import validate, ValidationError

   config = json.load(open("<config-file>"))
   schema = json.load(open("<schema-file>"))

   try:
       validate(instance=config, schema=schema)
   except ValidationError as e:
       # report e.message and e.path (which field failed)
   ```
   If `jsonschema` is not installed, install it (`pip install jsonschema`)
   or fall back to reporting that schema validation was skipped —
   never silently skip without telling the user.

5. **Report results** in this format:
   - `Syntax: OK` or `Syntax: FAILED — <error>`
   - `Schema: OK` / `Schema: FAILED — <field path>: <reason>` /
     `Schema: not checked (no schema found)`

## Example

Given `config.json` and `config.schema.json` in the same directory:

```
Syntax: OK
Schema: FAILED — properties.port: 'not-a-number' is not of type 'integer'
```
