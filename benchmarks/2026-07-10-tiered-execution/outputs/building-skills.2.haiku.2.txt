```markdown
# JSON Config Check

## Overview

This skill validates JSON configuration files for correct syntax and optionally validates them against a JSON Schema. Use this when you need to ensure configuration files are well-formed before deployment or when you want to verify they conform to an expected schema.

## When to Use

- Before deploying configuration files to catch syntax errors early
- To validate configuration files against a known schema (e.g., package.json, tsconfig.json)
- To ensure configuration files match your organization's required structure

## What It Does

The skill:
1. Validates that the JSON file has correct syntax
2. Optionally validates the file against a JSON Schema if one is present
3. Reports any parsing errors or schema violations with clear line/column information
4. Returns validation results indicating success or listing specific issues

## Input

- **Config file**: The JSON configuration file to validate
- **Schema file** (optional): A JSON Schema file in the same directory for additional validation

## Output

The skill returns:
- Valid/Invalid status
- List of errors (if any) with location information
- Validation details if a schema was used
```
