One thing worth flagging (not changing, since you asked me to follow the pattern): `like` with `"%@" + domain` treats `_` and `%` in the domain as wildcards, and it's case-insensitive by default — so `@it.example.com` also matches a query for `example.com`... wait, actually no: `"%@example.com"` requires the string to *end* effectively at `example.com` only if there's nothing after — `like` isn't anchored at the end unless the pattern ends there. `"%@example.com"` does anchor the end (no trailing `%`), so `foo@it.example.com` would need to end in `@example.com`, which it doesn't — good.
