We are setting up Claude Code for our repo 'checkout-service' and need a CLAUDE.md. Below is everything we know about the project. Write the finished file to CLAUDE.md in the current directory.

Project brief:
- Stack: Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.
- History: started as a Rails prototype in 2021, rewritten in 2023 by the platform team, now owned by the payments squad.
- Money: all amounts are integer cents; a float-rounding incident in 2024 corrupted invoices, so floats for money are banned.
- Tests: `npm test` runs the Playwright E2E suite and fails without a running dev server; unit tests are `npm run test:unit`.
- Migrations: always go through `./scripts/migrate.sh`; running `prisma migrate` directly breaks the shadow database our CI relies on.
- Auth: API routes authenticate with a custom `X-Session-Token` header scheme (not standard Bearer tokens); the full scheme is documented in docs/api.md.
- Style: our 45-page style guide lives at docs/style-guide.md; the only deviations from typical TypeScript style are: no default exports, and Zod schemas colocated next to their route handlers.
- Deploys (identical procedure used across all six company repos): 1. merge to main, 2. wait for CI green, 3. `npm run build:prod`, 4. `./deploy staging`, 5. smoke-test /healthz, 6. page on-call if p95 latency exceeds 500ms, 7. `./deploy prod`, 8. announce in #releases.
- Onboarding (already fully covered in README.md): clone, `npm install`, copy `.env.example` to `.env`, `npm run dev`.
