```markdown
# JSON Config Check

Validate a JSON configuration file: confirm it parses, and if a schema is
available, check the data against it. Report every error with its location.
Do not modify the file unless the user explicitly asks you to fix it.

## Steps

1. **Locate the target file.** Use the file named in the request. If none is
   named, search the working directory for likely configs (`*.json`,
   `config/*.json`) and confirm the target with the user before proceeding.

2. **Check well-formedness first.** Parse the file:
   `python3 -c "import json,sys; json.load(open(sys.argv[1]))" <file>`
   If parsing fails, report the exact line, column, and message from the
   error, then **stop** — schema validation on unparseable JSON is
   meaningless.

3. **Find a schema**, in this order:
   - a `$schema` reference inside the config file,
   - a sibling `<config>.schema.json` or `schema.json` in the same directory,
   - a path the user supplied.
   If no schema is found, report that the file is well-formed but was not
   schema-validated, then stop.

4. **Validate against the schema and collect all errors** (not just the
   first):
   ```
   python3 - "$CONFIG" "$SCHEMA" <<'PY'
   import json, sys
   from jsonschema import Draft7Validator
   data   = json.load(open(sys.argv[1]))
   schema = json.load(open(sys.argv[2]))
   errors = sorted(Draft7Validator(schema).iter_errors(data),
                   key=lambda e: list(e.path))
   for e in errors:
       path = "/".join(str(p) for p in e.path) or "(root)"
       print(f"{path}: {e.message}")
   sys.exit(1 if errors else 0)
   PY
   ```
   If `jsonschema` is missing, install it (`pip install jsonschema`) or tell
   the user it is required.

5. **Report results.** State pass/fail and an error count. For each error,
   give the JSON path (e.g. `server/port`), the failing value, and the
   constraint that was violated.

## Notes

- Report every schema error in one pass; do not stop at the first.
- Never write to the config file unless asked. If you do fix it, preserve
  key order and formatting.
- Match the schema draft to the config's `$schema` when one is declared;
  the snippet above defaults to Draft 7.
```
