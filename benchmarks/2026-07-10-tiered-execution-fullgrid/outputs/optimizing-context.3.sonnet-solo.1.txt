I don't have a Write tool available in this session (and the request explicitly says not to use any tools), so here are the decisions as text — treat this as the content for `placement.md`:

1. **Hook** (PostToolUse on file-edit tools) — a mandatory, zero-exception action tied to every edit event needs to be enforced mechanically, not left to model discretion.
2. **Skill** — a fixed, identical 10-step procedure reused across 14 repos is exactly the reusable, invoke-on-demand packaging a skill is for.
3. **Repo CLAUDE.md** — a repo-specific architectural convention that all contributors/agents in this codebase must follow belongs in the checked-in project instructions.
4. **MCP server** — looking up live external records (Salesforce) requires a live integration/connection, which is what MCP servers provide.
5. **Subagent** — a large, bounded, read-heavy investigative job (hundreds of manifests) is best isolated in its own context so it doesn't bloat the main conversation, then reports back a summary.
6. **User-level CLAUDE.md / user preferences** — this is Dana's personal preference that follows her across every project, not something tied to any one repo.
7. **Repo CLAUDE.md** — current schema/migration state is project state that should be versioned and updated alongside migrations so it stays authoritative for anyone (or any agent) working in the repo.
8. **Plain one-off prompt** — a single, non-recurring mechanical rename doesn't warrant building any reusable mechanism.
