```markdown
# Claude Instructions

## API Auth

**CRITICAL:** Requests to `/api/*` must carry an `X-Client-Sig` header.

- Header value: HMAC-SHA256 of the request body, signed with the key from `scripts/sign.mjs`
- This is **NOT** standard Bearer auth or Authorization-header authentication
- Do not write Bearer tokens or Authorization headers for API calls — they will fail
- For full details, see `docs/auth.md`

## Style

- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase; hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## Tests

- `pnpm test:unit` — use this for iteration; runs fast, no external dependencies
- `pnpm test` — launches real Chromium instances and requires the dev server running; do not use by default for quick feedback

## Onboarding

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open localhost:3000

## Deploy Runbook

1. Confirm main is green
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB
4. Tag release vX.Y.Z
5. `./ship staging`
6. Wait for staging healthcheck
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242
9. Post staging sign-off in #launch
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; if error rate exceeds 0.5%, rollback via `./ship rollback`
12. `./ship prod --full`
13. Verify Sentry release tagging
14. Announce in #releases
```
