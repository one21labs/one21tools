---
name: json-config-check
description: Validates JSON configuration files for syntax correctness and, when a matching JSON Schema is available, schema compliance. Use when the user asks to check, validate, or lint a JSON config file, debug a JSON parse error, or verify a config matches its schema.
---

# JSON Config Check

## Purpose
Validate that a JSON configuration file is syntactically valid, and — if a corresponding JSON Schema is available — that it conforms to that schema.

## Procedure

1. Locate the target JSON file (path given by the user, or inferred from context).
2. Parse it:
   ```python
   import json
   with open(path) as f:
       data = json.load(f)
   ```
   If parsing fails, report the `json.JSONDecodeError` message verbatim, including line/column. Do not guess at a fix without showing the user the actual error.
3. Look for a schema, in this order:
   - A schema file explicitly given by the user
   - `<config-name>.schema.json` in the same directory
   - A `$schema` key inside the JSON file pointing to a local path
   If none is found, report the file as syntactically valid and stop. Do not fabricate a schema.
4. If a schema is found, validate with `jsonschema`:
   ```python
   from jsonschema import Draft7Validator
   validator = Draft7Validator(schema)
   errors = sorted(validator.iter_errors(data), key=lambda e: e.path)
   ```
   Collect *all* errors, not just the first — report each error's message and JSON path.
5. If `jsonschema` isn't installed, don't fail silently: tell the user schema validation is unavailable and report syntax-only results.
6. Summarize to the user: valid/invalid, and the full list of errors (if any).

## Notes
- A syntactically valid JSON file may be a bare array, string, or number, not just an object — don't treat that as an error unless the schema requires an object.
- Don't attempt to auto-fix the config; report findings only unless the user explicitly asks for fixes.
```
