# CLAUDE.md

Guidance for Claude Code when working in checkout-service.

## Project

Next.js 14 (App Router) + TypeScript + Tailwind, Prisma on Postgres. Owned by the payments squad. Standard npm workflow.

Onboarding is fully covered in README.md — follow that, don't duplicate it here.

## Critical guardrails

**Money is always integer cents. Never use floats for money.** A float-rounding bug corrupted invoices in 2024 — this is a hard rule, not a style preference.

**Never run `prisma migrate` directly.** It breaks the shadow database CI relies on. Always go through `./scripts/migrate.sh`.

**API auth uses a custom `X-Session-Token` header, not Bearer tokens.** Full scheme is documented in docs/api.md — check it before writing or modifying auth code.

## Tests

- `npm test` runs the Playwright E2E suite and requires a running dev server. It will fail if one isn't up — that's not necessarily a sign the code is broken.
- `npm run test:unit` runs unit tests — use this for quick verification.

## Style

Full style guide: docs/style-guide.md (45 pages). Only two deviations from typical TypeScript conventions:

- No default exports.
- Zod schemas live next to the route handlers that use them.

## Deploys

Same procedure across all six company repos:

1. Merge to main
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in #releases

These steps affect shared/production systems — confirm with the user before executing staging/prod deploy commands rather than running them autonomously.
