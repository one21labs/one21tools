```markdown
# CLAUDE.md

Guidance for working in **checkout-service**, owned by the payments squad.

## Stack

Next.js 14 (App Router) · TypeScript · Tailwind · Prisma on Postgres. Standard npm workflow.

Onboarding (clone, install, env, dev server) is in **README.md** — start there.

## Critical rules

These are the ways to get things wrong that have bitten us before. Follow them exactly.

### Money is always integer cents — never floats
All monetary amounts are stored and computed as **integer cents**. Do not introduce
floating-point numbers for money anywhere, including intermediate calculations. A
float-rounding bug corrupted invoices in 2024; floats for money are banned.

### Migrations go through the wrapper script
Always run database migrations via:
```
./scripts/migrate.sh
```
Never run `prisma migrate` directly — it breaks the shadow database that CI depends on.

## Testing

- `npm test` — Playwright **E2E** suite. Requires a running dev server; it will fail
  without one.
- `npm run test:unit` — unit tests (no dev server needed).

## Code style

The full style guide is **docs/style-guide.md** (read it for anything non-obvious).
Two deviations from typical TypeScript style are worth calling out because tooling
won't always catch them:

- **No default exports** — use named exports only.
- **Colocate Zod schemas** next to their route handlers.

## Auth

API routes use a custom **`X-Session-Token`** header scheme — *not* standard Bearer
tokens. The full scheme is documented in **docs/api.md**; consult it before touching
auth-related code.

## Deploys

Standard company deploy procedure (identical across all six repos):

1. Merge to `main`
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. If p95 latency exceeds 500ms, page on-call
7. `./deploy prod`
8. Announce in **#releases**
```
