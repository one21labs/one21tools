## `.git/hooks/pre-commit`

```sh
#!/bin/sh
if git diff --cached | grep -E '^\+' | grep -v '^\+\+\+' | grep -q 'DO NOT COMMIT'; then
    echo "Commit rejected: staged changes contain 'DO NOT COMMIT'." >&2
    exit 1
fi
```

Save that as `.git/hooks/pre-commit` and `chmod +x .git/hooks/pre-commit`. It only scans *added* lines in the staged diff (skipping the `+++` file-header line), so removing a line containing the marker won't falsely block a commit.

## Verify it yourself

```sh
mkdir /tmp/hook-demo && cd /tmp/hook-demo && git init -q

# install the hook
mkdir -p .git/hooks
cat > .git/hooks/pre-commit <<'EOF'
#!/bin/sh
if git diff --cached | grep -E '^\+' | grep -v '^\+\+\+' | grep -q 'DO NOT COMMIT'; then
    echo "Commit rejected: staged changes contain 'DO NOT COMMIT'." >&2
    exit 1
fi
EOF
chmod +x .git/hooks/pre-commit

# 1. clean commit — should succeed
echo "hello world" > a.txt
git add a.txt
git commit -m "add a.txt"
```

Expected: normal commit output, e.g. `[main (root-commit) abc1234] add a.txt`.
