# Project Guidelines

## About This Project
This is a Python web application built with Flask. Flask is a lightweight WSGI micro web framework for Python that is easy to get started with. We chose it for its simplicity and flexibility. The app serves an internal orders dashboard.

## Code Style
We write Python 3.11. Python is an interpreted, high-level, general-purpose programming language. Always use 4 spaces for indentation as per PEP 8. Use meaningful variable names. Write docstrings for public functions. Keep functions short and focused. Prefer f-strings over .format().

## Getting Started
Clone the repository, create a virtualenv with `python -m venv .venv`, activate it, run `pip install -r requirements.txt`, copy `.env.example` to `.env`, then start the server with `flask run`.

## Database
The orders table schema is currently at version 12 (defined in models.py). Never use raw SQL.

## Caching
The /reports endpoint caches its aggregates in Redis with a 15-minute TTL; if report numbers look stale, flush the `reports:*` keys.

## Conventions That Matter
- Use `datetime.now(tz=timezone.utc)` everywhere; a 2023 bug from naive datetimes double-charged customers in CET.
- Run `make lint` before committing; CI runs the exact same target.
- Local tests: `pytest -m "not integration"`; the integration marker needs the docker compose stack up.
