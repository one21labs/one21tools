def handle_invoice_event(ledger, payload):
    if payload["event_type"] == "invoice.paid":
        invoice = payload["invoice"]
        ledger[invoice["id"]] = invoice["amount_cents"]

One thing worth flagging separately from input-shape validation: does anything upstream of this handler verify the webhook's authenticity (e.g., an HMAC signature check on the raw request body)? "The provider validates the event" usually means *they* checked the data is well-formed before sending it — it doesn't mean *you've* confirmed the POST actually came from them and not a forged request hitting your endpoint URL. If that verification isn't already handled in middleware/routing before `payload` reaches this function, that's the gap I'd want closed, not the dict-shape checks your tech lead had in mind.
