Here's a portable bash script you can drop into CI (e.g. as `scripts/check-file-length.sh`):

```bash
#!/usr/bin/env bash
# Fails the build if any file under src/ exceeds MAX_LINES lines.
set -euo pipefail

MAX_LINES=500
SRC_DIR="src"

if [[ ! -d "$SRC_DIR" ]]; then
    echo "error: '$SRC_DIR' directory not found" >&2
    exit 1
fi

failed=0

while IFS= read -r -d '' file; do
    lines=$(wc -l < "$file")
    if (( lines > MAX_LINES )); then
        echo "FAIL: $file has $lines lines (max $MAX_LINES)"
        failed=1
    fi
done < <(find "$SRC_DIR" -type f -print0)

if (( failed )); then
    echo ""
    echo "One or more files exceed the $MAX_LINES line limit."
    exit 1
fi

echo "All files under $SRC_DIR/ are within the $MAX_LINES line limit."
exit 0
```

Make it executable: `chmod +x scripts/check-file-length.sh`

Usage in CI (e.g. GitHub Actions step):
```yaml
- name: Check file length
  run: ./scripts/check-file-length.sh
```

Notes:
- Checks *all* files under `src/`, not just a specific extension — if you want to restrict it (e.g. only `.ts`/`.py`), tell me the extensions and I'll add a `-name` filter to the `find` call.
- `wc -l` counts newlines, so a file with a final line lacking a trailing newline will be undercounted by 1 — usually not worth working around, but flag it if you need exact counts.
