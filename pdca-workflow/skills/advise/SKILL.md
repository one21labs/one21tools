# /advise — the advisor panel, standalone (Plan input)

The dialectic half of the `/decide` panel as a right-sized primitive: run it when a call needs
argued perspectives but not (or not yet) the full ceremony. The main agent or the user picks it
situationally; `/decide` composes it and adds the record.

## Run

1. **Frame.** Resolve the ask into ONE stated scope line before any advisor runs — a fuzzy ask
   wastes spend and anchors wrong.
2. **Pick the panel.** Use the project's advisor panel in `.claude/agents/` (generated by
   `/pdca-init`); pick the 2-3 advisors this call needs and note why that subset. If the named
   agents are not spawnable as registered types (session cwd outside the repo root), spawn
   general-purpose agents instructed to read and adopt `.claude/agents/<name>.md` verbatim —
   never the generic fallback while a panel exists. No panel = spawn two generic opposing
   counsel. A genuinely two-sided call always gets opposing counsel (each steelmans one side).
3. **Spawn fresh, in parallel, never primed** — no target grade, no hint of the asker's
   preferred answer. Pass each only the files it needs.
4. **Require the shape:** a recommendation + effort x risk x value grounded in current
   code/output (cite file:line) + THE one assumption it depends on, tagged per
   `../decide/references/adr-template.md` — terse.

## Return

Hand the recommendations back verbatim — advisors advise; the CALLER decides. Unanimous
consensus either way is a mis-scope signal: say so and re-confirm the ask. If the call must be
recorded (roadmap, product, policy), escalate to `/decide` — deciding without a record is
drift.
