Separately, worth flagging before you do this yourself: restating the same number (30s) in three docs plus the code is four places that can drift out of sync the next time someone changes the timeout. A tighter approach — one fact, lowest-altitude owner:

- Treat `config.py` as the source of truth for the actual value.
- Document it in **one** doc at the altitude that owns "how to configure the client" (probably the API guide, since that's where request behavior belongs).
- Have the README and onboarding doc *reference* that section ("see API guide § Timeouts") rather than restating "30s."

If a doc absolutely needs the number inline for readability (e.g., onboarding), keep it, but I'd still point back to the API guide as canonical rather than triplicating prose.
