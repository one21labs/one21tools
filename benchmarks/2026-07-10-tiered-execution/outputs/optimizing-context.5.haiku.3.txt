# CLAUDE.md

## Code Style

- **Prefer `const` over `let`.** Use meaningful, descriptive names.
- **Component naming:** PascalCase for components, `use` prefix for hooks.
- **Keep functions small.** Avoid deeply nested ternaries.

## API Authentication

All requests to `/api/*` endpoints must include an `X-Client-Sig` header—an HMAC-SHA256 of the request body using the key from `scripts/sign.mjs`.

**This is custom authentication, not Bearer auth.** See `docs/auth.md` for implementation details. Claude frequently defaults to standard Bearer token patterns here and fails; verify you're using HMAC-SHA256 with the signature key.

## Testing

- **Integration/e2e tests:** `pnpm test` launches real Chromium instances and requires the dev server running.
- **Unit tests:** `pnpm test:unit` for fast iteration without starting the dev server.

## Onboarding

1. Clone the repo.
2. Run `pnpm install`.
3. Copy `.env.example` to `.env`.
4. Run `pnpm dev`.
5. Open `localhost:3000`.

## Deployment Runbook

Follow these 14 steps in order:

1. **Confirm main is green** — ensure CI passes on the main branch.
2. **Build:** `pnpm build`
3. **Bundle analysis:** `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. **Tag release:** `git tag vX.Y.Z` (follow semantic versioning).
5. **Ship to staging:** `./ship staging`
6. **Wait for staging healthcheck** to complete (watch logs).
7. **Run smoke suite:** `pnpm smoke:staging`
8. **Manual verification:** Test checkout flow with test card `4242`.
9. **Post sign-off:** Post staging sign-off message in `#launch` channel.
10. **Ship to production (canary):** `./ship prod --canary 5` (5% traffic).
11. **Monitor error rate** for 15 minutes. If error rate exceeds 0.5%, run `./ship rollback` immediately.
12. **Full production rollout:** `./ship prod --full`
13. **Verify Sentry:** Confirm release is tagged correctly in Sentry.
14. **Announce:** Post release announcement in `#releases` channel.
