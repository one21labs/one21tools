# CLAUDE.md

## Org-wide Conventions

- **Commits:** Conventional commits required (`fix:`, `feat:`, `refactor:`, etc.) — enforced in CI.
- **Pre-PR gate:** `make check` must pass locally before any PR submission.
- **Development model:** Trunk-based development only — no long-lived feature branches.

## Service Navigation

This is a monorepo with three independent services. Each has its own `CLAUDE.md` with service-specific conventions and quirks:

- **`api/`** (Go, gRPC) — see `api/CLAUDE.md` for proto generation and api/gen/ rules.
- **`web/`** (React, TypeScript) — see `web/CLAUDE.md` for build, test, and Playwright distinctions.
- **`pipeline/`** (Python, Airflow) — see `pipeline/CLAUDE.md` for DAG import prerequisites and AIRFLOW_HOME setup.

Claude Code will load service-specific rules automatically when you open files in those directories.

## Incident Response

All engineers (across the company) use the same 20-step incident-response runbook.

@runbooks/incident-response.md

Ask Claude to walk you through it by name or step reference; the runbook is in your context when you need it.

## Analytics & SQL Access

Live SQL queries against the analytics warehouse are available via Claude's analytics integration (configured as an MCP server or skill at the workspace level). When asked about product metrics, Claude can execute read-only queries directly. Refer to workspace setup docs for credentials and permission scoping.
