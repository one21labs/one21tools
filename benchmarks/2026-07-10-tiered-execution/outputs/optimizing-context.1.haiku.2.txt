## Stack & Architecture

- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript (strict mode)
- **Styling:** Tailwind CSS
- **Database:** Postgres + Prisma ORM
- **Package Manager:** npm

Originated as a Rails prototype (2021), rewritten 2023 by platform team, now maintained by payments squad.

## Critical Constraints

### Money Handling: Integer Cents Only

**DO NOT USE FLOATS FOR MONETARY AMOUNTS.** All money is stored and calculated as integer cents. A float-rounding bug in 2024 corrupted invoices—this is not a style preference, it is a hard requirement.

- `100` = $1.00, `1050` = $10.50
- Never divide money by anything that could produce a decimal
- Never store money as a float in the database
- Validate at the boundary: reject decimal input and convert to cents

## Database & Migrations

**Always use `./scripts/migrate.sh`** for all schema changes. Do not run `prisma migrate` directly—it breaks the shadow database that CI depends on.

```bash
./scripts/migrate.sh  # runs migrations safely
```

The migration script handles shadow DB synchronization; direct Prisma runs will cause CI to fail.

## Testing

- **Unit tests:** `npm run test:unit` (can run anytime)
- **E2E tests:** `npm test` (requires `npm run dev` to be running in another terminal)

The E2E suite uses Playwright and depends on a live dev server. Both must pass before merging to main.

## Authentication

API routes use a custom `X-Session-Token` header scheme, **not** standard Bearer tokens.

Full authentication scheme and token format are documented in `docs/api.md`. Always reference that file when:
- Adding new API routes
- Validating tokens
- Debugging auth failures

## Code Style

Follow the **45-page style guide** at `docs/style-guide.md`. Two key deviations from typical TypeScript:

1. **No default exports.** Use named exports only.
2. **Colocate Zod schemas** with their route handlers (same file, no separate schema modules).

## Deployment

Identical procedure across all six company repos:

1. Merge to `main`
2. Wait for CI to go green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz` endpoint
6. If p95 latency exceeds 500ms, page on-call; do not proceed to prod
7. `./deploy prod`
8. Announce in `#releases` Slack channel

Do not skip steps or proceed if health checks fail.
