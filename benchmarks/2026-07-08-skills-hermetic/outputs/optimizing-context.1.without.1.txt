# CLAUDE.md

Guidance for Claude Code when working in checkout-service.

## Project
Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard npm workflow.
Originally a 2021 Rails prototype, rewritten in 2023 by the platform team. Currently owned by the payments squad.

## Money handling — no floats, ever
All monetary amounts are stored and manipulated as **integer cents**. Never use floats for money.
A 2024 incident where float rounding corrupted invoices is why this is a hard rule, not a style preference. Treat any diff introducing a float for a price/amount/total as a bug.

## Tests
- `npm test` — Playwright E2E suite. Requires a running dev server (`npm run dev` in another process) or it fails outright. Don't run it as a plain correctness check without one running.
- `npm run test:unit` — unit tests, no server required.

## Database migrations
Always create/run migrations via `./scripts/migrate.sh`. Never call `prisma migrate` directly — it breaks the shadow database CI depends on.

## Auth
API routes authenticate via a custom `X-Session-Token` header scheme — not standard `Authorization: Bearer`. Full scheme documented in `docs/api.md`; consult it before touching auth-related route code.

## Style
Full style guide: `docs/style-guide.md`. It's mostly standard TypeScript style; the notable deviations are:
- No default exports.
- Zod schemas are colocated next to their route handlers, not centralized.

## Deploys
Same procedure as the other five company repos:
1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. Page on-call if p95 latency exceeds 500ms.
7. `./deploy prod`.
8. Announce in `#releases`.

Deploy steps affect shared/production systems — confirm with the user before running any `./deploy` command rather than executing the sequence autonomously.

## Onboarding
See `README.md` (clone, `npm install`, copy `.env.example` to `.env`, `npm run dev`).
