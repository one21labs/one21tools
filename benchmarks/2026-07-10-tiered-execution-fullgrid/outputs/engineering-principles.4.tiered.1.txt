# Config Subsystem Completion Plan

## Scope Check

**Actual requirement (from sole consumer):** Read `config.json` at startup and validate three fields: `api_url`, `team_id`, `log_level`. This is complete or nearly complete via the JSON adapter.

**In-flight components with no explicit consumer request:**
- YAML, TOML, XML adapters — no team has asked for multi-format support
- Hot-reload watcher — startup-only validation was the ask, not runtime reloads
- Migration framework — no versioned config upgrades requested

Two weeks of sunk cost on the pluggable architecture is real, but it's separate from the decision to finish it. The work below assumes the team has decided to complete these components anyway; this plan is scoped to land the highest-value pieces first within the sprint.
