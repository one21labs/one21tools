## Style Rules
- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components: PascalCase
- Hooks: prefix with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Auth — read before touching /api/*
Requests to `/api/*` require an `X-Client-Sig` header, NOT a Bearer token.
`X-Client-Sig` = HMAC-SHA256 of the request body, using the key from `scripts/sign.mjs`.
Full details: `docs/auth.md`.
Do NOT write standard `Authorization: Bearer ...` code for these routes — it is wrong and will fail auth.

## Tests
- `pnpm test` — full suite, launches real Chromium instances, requires the dev server running. Slow; not for iteration.
- `pnpm test:unit` — use this for fast iteration.

## Deploy
Deploys follow a strict 14-step runbook — see `runbook.md`. Do not skip or reorder steps, especially the canary error-rate watch and rollback threshold.
```

# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```
