# CLAUDE.md

Guidance for Claude Code when working in this repo.

## Project

`checkout-service` — owned by the payments squad. Originally a Rails prototype (2021),
rewritten by the platform team in 2023. Next.js 14 (App Router), TypeScript, Tailwind,
Prisma on Postgres.

Onboarding is fully covered in README.md (clone, `npm install`, copy `.env.example` to
`.env`, `npm run dev`) — don't duplicate it here.

## Money handling

**All monetary amounts are integer cents. Never use floats for money.** A float-rounding
bug corrupted invoices in 2024. Watch for this in new code and flag it in review.

## Testing

- `npm test` runs the Playwright E2E suite. It requires a running dev server — start one
  first, or it will fail.
- `npm run test:unit` runs unit tests and does not need a dev server.

## Database migrations

Never run `prisma migrate` directly — it breaks the shadow database CI relies on.
Always go through `./scripts/migrate.sh`.

## Auth

API routes authenticate via a custom `X-Session-Token` header scheme — not standard
Bearer tokens. Full scheme is documented in `docs/api.md`; consult it before touching
auth code.

## Style

Full style guide: `docs/style-guide.md`. Deviations from typical TypeScript conventions:
- No default exports.
- Zod schemas are colocated next to their route handlers.

## Deploy procedure

Same procedure across all six company repos:

1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. Page on-call if p95 latency exceeds 500ms.
7. `./deploy prod`.
8. Announce in `#releases`.

Do not skip steps or deploy straight to prod.
