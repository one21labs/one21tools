# JSON Config Check

## Purpose

Use this skill when you need to validate JSON configuration files for syntax errors or schema compliance before deployment. This is critical for catching configuration mistakes early.

## What It Does

The skill reads a JSON file and validates it in two ways:

1. **Syntax validation**: Checks that the file is valid JSON (proper braces, quotes, commas, escaping).
2. **Schema validation**: If a `schema.json` file exists in the same directory as the config file, validates the config against that schema and reports any violations.

## How to Use

Provide the path to a JSON configuration file. The skill will:
- Parse the JSON and report any syntax errors with line numbers and descriptions.
- If a schema file is present, validate the config against it and report schema violations (missing required fields, type mismatches, etc.).
- Return a summary of all errors found, or confirm the file is valid.

## Example

**Input:** Path to `config.json` in a directory that contains `schema.json`

**Output:** Either a validation error ("Missing required field 'database.host' at line 12") or success ("Valid JSON and passes schema validation").

## Implementation Notes

Uses Python's `json` module for syntax validation and `jsonschema` package for schema-based validation. Both are standard in most Python environments.
