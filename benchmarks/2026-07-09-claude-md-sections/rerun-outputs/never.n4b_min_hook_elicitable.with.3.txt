One clarification: a **`pre-commit`** hook runs before you even type a commit message, so it can't see the message text. To reject commits based on message content, you need a **`commit-msg`** hook (it receives the message file path as `$1`). Here's the minimal version:

**`.git/hooks/commit-msg`**
```bash
#!/usr/bin/env bash
grep -q 'DO NOT COMMIT' "$1" && { echo "commit-msg: rejected — message contains 'DO NOT COMMIT'"; exit 1; }
```

Make it executable:
```bash
chmod +x .git/hooks/commit-msg
```

Verify both cases:
```bash
git commit --allow-empty -m "clean commit message"
# expected: commits normally

git commit --allow-empty -m "DO NOT COMMIT this one"
# expected:
# commit-msg: rejected — message contains 'DO NOT COMMIT'
```
