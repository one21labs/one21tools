# JSON Config Check

## Procedure

1. **Locate the config file**: The caller will provide the path to a JSON configuration file to validate.

2. **Locate the schema file**: Look for a schema file in the same directory as the config file. Accepted names are `schema.json` or any file matching the pattern `*.schema.json`. If multiple schema files exist, use the first match in lexicographic order.

3. **If no schema file is found**: Parse the JSON config file for syntax validity only. Report the result to the user (see Output below).

4. **If a schema file is found**:
   - Parse both the config file and the schema file as JSON.
   - Validate the config against the schema using a standard JSON Schema validator (e.g., Python's `jsonschema` library with `jsonschema.validate(config, schema)`).
   - Report the result to the user (see Output below).

## Error Handling

- **JSON parse error** (malformed JSON): Report file path, line number, character position, and the parse error message. Example: `Config file error at line 5, col 12: Unexpected token 'x'`.

- **Schema validation error** (valid JSON that doesn't conform to schema): Report the failing field path, expected type/constraint, and the actual value. Example: `Validation failed for field 'port': expected integer, got string "8080"`.

- **Schema file parse error**: Report file path and parse error. Stop validation and inform the user the schema is malformed.

## Output

- **On success**: Report that the config file is valid. If a schema was used, state: `Config is valid against schema`. If no schema was found, state: `Config is valid JSON`.

- **On failure**: List all validation errors encountered (parse errors and/or schema violations), with file paths and specific locations. Do not proceed with further processing.
