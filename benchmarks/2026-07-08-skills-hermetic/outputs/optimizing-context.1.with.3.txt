```markdown
# checkout-service

Next.js 14 (App Router) + TypeScript + Tailwind + Prisma/Postgres. Owned by the payments squad. Standard npm workflow.

## Money handling
All monetary amounts are integer cents. **Never use floats for money** — a 2024 float-rounding bug corrupted invoices. If you see a float being used for a price/amount, flag it.

## Tests
- `npm test` runs the Playwright E2E suite — requires a running dev server, or it fails.
- `npm run test:unit` runs unit tests.

## Migrations
Always run migrations through `./scripts/migrate.sh`. Never run `prisma migrate` directly — it breaks the shadow database CI depends on.

## Auth
API routes authenticate via a custom `X-Session-Token` header, not standard Bearer tokens. Full scheme: docs/api.md.

## Style
Full style guide: docs/style-guide.md. Two deviations from typical TypeScript conventions worth remembering:
- No default exports.
- Zod schemas are colocated next to their route handlers.

## Deploys
Same procedure across all six company repos:
1. Merge to main
2. Wait for CI green
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`
6. Page on-call if p95 latency exceeds 500ms
7. `./deploy prod`
8. Announce in #releases

Deploy steps (staging/prod pushes, paging on-call) are high-blast-radius — confirm with the user before executing them.
```
