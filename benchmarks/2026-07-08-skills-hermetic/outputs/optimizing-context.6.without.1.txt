```markdown
# CLAUDE.md

## Code Style
- Use 4-space indentation in Python files.
- Follow PEP 8.
- Prefer f-strings over `.format()`.
- Use descriptive variable names.
- Add docstrings to all public functions.
- Keep functions short and focused.
- Prefer composition over inheritance.
- Do not use mutable default arguments.
- Prefer early returns over deep nesting.

## Error Handling & Logging
- Handle exceptions; do not let them crash silently.
- Error messages should include context about what failed.
- Use logging, not print statements.

## Testing
- Write unit tests for new code.
- Run tests locally with `poetry run pytest -m "not slow"`. Tests marked `slow` require the seeded DB from `make db-seed`.

## Git & Pull Requests
- Never commit secrets to the repository.
- Keep pull requests small.
- Branch names use the ticket prefix, e.g. `PLAT-123-short-description`.

## Build & Commit Checks
- Format code before committing.
- Run `make check` before every commit; CI runs the identical target.

## HTTP & Retries
- All outbound HTTP goes through the wrapper in `core/http.py`; raw `requests` calls bypass our retry budget.
- Retries are capped at 3 attempts.

## Data & Schema
- The events table schema is at version 9.
- Config keys are `API_URL`, `RETRY_MAX`, `QUEUE_NAME`, and `BATCH_SIZE` (defined in `settings.py`).
- Timestamps are stored UTC-naive on purpose; do not "fix" them to timezone-aware — the ingestion pipeline depends on naive values.

## Generated Code
- Never edit files under `gen/` by hand; they are regenerated by `make codegen`.

## Migrations
- Migrations run only via `make migrate`; invoking `alembic` directly skips the tenant loop.

## Feature Flags
- ⚠️ **Needs update:** Previously — "Check flags with `flags.is_enabled()` before shipping gated features." The feature-flag service was decommissioned in Q1 2026 and `flags.py` no longer exists. Confirm whether a replacement mechanism exists for gated features, or remove this rule.

## Deployment
- Production deploys happen only from the main branch.

## Documentation
- Record non-obvious decisions in `docs/decisions/`.
```
