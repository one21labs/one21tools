## Style Rules

- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase, hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Auth (READ BEFORE WRITING /api/* CODE)

**All requests to `/api/*` must include an `X-Client-Sig` header.**

This header is **NOT** a Bearer token or Authorization header — this is custom HMAC authentication.

- The `X-Client-Sig` header is an HMAC-SHA256 signature of the request body
- Sign the body using the key from `scripts/sign.mjs`
- **Do NOT use Bearer auth or Authorization header** — that authentication method will always fail here

See `docs/auth.md` for full details on implementing X-Client-Sig signing.

## Tests

- `pnpm test` launches real Chromium instances and requires the dev server to be running
- For fast iteration during development, use `pnpm test:unit` instead (does not require the dev server)
- Use `pnpm test` for full integration testing; use `pnpm test:unit` for quick unit test feedback loops

## Onboarding

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open `localhost:3000`

## Deploy Runbook

1. Confirm main is green
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB
4. Tag release `vX.Y.Z`
5. `./ship staging`
6. Wait for staging healthcheck to complete
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card `4242`
9. Post staging sign-off in #launch
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if error rate > 0.5%
12. `./ship prod --full`
13. Verify Sentry release tagging
14. Announce in #releases
