## Overview

This skill validates JSON configuration files against a JSON Schema definition. Use it to catch malformed JSON, missing required fields, or values that violate schema constraints before deploying configurations.

## When to Use

- Validating configuration files match expected structure
- Catching JSON syntax errors early
- Ensuring required fields are present
- Detecting type mismatches in config values

## How to Invoke

Provide the skill with:
1. A JSON configuration file to validate
2. Optionally, a JSON Schema file in the same directory (if present, validation runs automatically)

## Example

**Input:** `config.json` with optional `config.schema.json`

- Error list with specific violations (if validation fails)
