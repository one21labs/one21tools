# JSON Config Check

## Purpose

This skill validates JSON configuration files for correctness, checking both syntax and schema compliance.

## How to Use

1. Locate the JSON configuration file to validate.
2. Check if a JSON schema file exists in the same directory (e.g., `schema.json`, `config.schema.json`).
3. Parse the configuration file using Python's `json` module. Report any parse errors with line and column numbers.
4. If a schema file is present, validate the parsed configuration against it using the `jsonschema` package. Report all schema violations.
5. Report success if parsing succeeds and validation (if applicable) passes.

## Tools

Use Python 3 with:
- `json` module (built-in) for parsing
- `jsonschema` package for schema validation

## Example Output

**Success (no schema present):**
```
✓ config.json is valid JSON
```

**Success (with schema):**
```
✓ config.json is valid JSON
✓ Matches schema.json
```

**Failure (parse error):**
```
✗ config.json has invalid JSON at line 5, column 12: Expecting ',' delimiter
```

**Failure (schema validation):**
```
✓ config.json is valid JSON
✗ Schema validation failed:
  - "port" is required
  - "timeout" must be an integer, got string
```
