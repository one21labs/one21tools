```markdown
# checkout-service

Next.js 14 (App Router), TypeScript, Tailwind, Prisma/Postgres. Owned by the payments squad. Standard npm workflow. Originally started as a Rails prototype in 2021; rewritten in 2023 by the platform team.

See [README.md](README.md) for onboarding (clone, install, `.env.example` → `.env`, `npm run dev`).

## Money Handling

**All amounts are integer cents; floats for money are banned.** A float-rounding incident in 2024 corrupted invoices. Never use floats for monetary values, even in intermediate calculations.

## Testing

- `npm test` runs the Playwright E2E suite. **Requires a running dev server first** — tests will fail if the server is not already running.
- `npm run test:unit` runs unit tests (no server needed).

## Migrations

Always use `./scripts/migrate.sh`. Never run `prisma migrate` directly — it breaks the shadow database that CI relies on.

## Auth

API routes authenticate with a custom `X-Session-Token` header scheme, not standard Bearer tokens. See [docs/api.md](docs/api.md) for the full scheme.

## Style

Reference [docs/style-guide.md](docs/style-guide.md) for the canonical style guide. The only deviations from typical TypeScript style are:

- No default exports.
- Zod schemas colocated next to their route handlers.

## Deploy

The following procedure is used identically across all six company repos:

1. Merge to main.
2. Wait for CI to pass.
3. `npm run build:prod`
4. `./deploy staging`
5. Smoke-test `/healthz`.
6. If p95 latency exceeds 500ms, page on-call.
7. `./deploy prod`
8. Announce in #releases.
```
