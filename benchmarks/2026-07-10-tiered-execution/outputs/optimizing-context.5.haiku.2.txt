# Development Guide

## Quick Start (Onboarding)

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open localhost:3000

## Code Style

- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase
- Hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## Testing

- `pnpm test` launches real Chromium instances and requires the dev server running
- For iteration, use `pnpm test:unit` (faster, no server dependency)

## API Authentication

All requests to `/api/*` must carry an `X-Client-Sig` header. This is **not** Bearer auth—it's a custom HMAC-SHA256 signature of the request body using the key from `scripts/sign.mjs`.

**Critical:** Claude consistently writes standard Bearer code here and it always fails. Refer to `docs/auth.md` for full details.

5. Wait for staging healthcheck

7. Verify checkout flow manually with test card 4242
