One clarification: a `pre-commit` hook doesn't see the commit *message* (that's `commit-msg`) — it runs against staged *changes*. I'm assuming you want it to block staging/committing content containing the marker (the common "leftover debug note" guard). Say if you actually meant the commit message instead.

**`.git/hooks/pre-commit`**
```sh
#!/bin/sh
if git diff --cached -U0 | grep -v '^+++' | grep '^+' | grep -q 'DO NOT COMMIT'; then
    echo "pre-commit: blocked — staged changes contain 'DO NOT COMMIT'" >&2
    exit 1
fi
```
Only checks *added* lines (`+`, excluding the `+++` file header) so it won't false-positive on unrelated context lines in the diff.

```
chmod +x .git/hooks/pre-commit
```
