## Project Overview

`checkout-service` is owned by the **payments squad**. It started as a Rails prototype in 2021 and was rewritten in 2023 by the platform team into its current form.

**Stack:** Next.js 14 (App Router), TypeScript, Tailwind, Prisma on Postgres. Standard `npm` workflow.

## Onboarding

Fully covered in `README.md` (clone, `npm install`, copy `.env.example` to `.env`, `npm run dev`). Don't duplicate that here — refer to the README.

## Critical Rule: Money Handling

**All monetary amounts are integer cents. Floats must never be used to represent money.**

A float-rounding bug in 2024 corrupted invoices in production. This is a hard rule, not a style preference — reject any code that stores or computes money as a float.

## Testing

- `npm test` — runs the Playwright E2E suite. **Requires a running dev server**; it will fail without one.
- `npm run test:unit` — unit tests, no dev server required.

## Database Migrations

Always run migrations through `./scripts/migrate.sh`.

**Never run `prisma migrate` directly** — it breaks the shadow database that CI relies on.

## Auth

API routes authenticate using a custom `X-Session-Token` header scheme — **not** standard Bearer tokens. Full scheme details are documented in `docs/api.md`; consult that doc before implementing or modifying auth logic.

## Style

Full style guide: `docs/style-guide.md` (45 pages). Notable deviations from typical TypeScript style:

- No default exports.
- Zod schemas are colocated next to their route handlers.

## Deploy Procedure

Same procedure used across all six company repos:

1. Merge to `main`.
2. Wait for CI green.
3. `npm run build:prod`.
4. `./deploy staging`.
5. Smoke-test `/healthz`.
6. Page on-call if p95 latency exceeds 500ms.
7. `./deploy prod`.
8. Announce in `#releases`.
