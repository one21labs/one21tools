```markdown
# Claude Code Mechanism Placement Decisions

1. **Hook** — a deterministic "always run, no exceptions" post-edit action is exactly what a PostToolUse hook enforces; relying on the model to remember is not guaranteed.
2. **Skill** — an identical, on-demand multi-step procedure shared across 14 repos belongs in one reusable skill rather than duplicated repo CLAUDE.md text.
3. **Repo CLAUDE.md** — a project-specific coding convention that should passively inform every Claude session working in this repo.
4. **MCP server** — looking up live records in an external system (Salesforce) requires a live data connection, not static context.
5. **Subagent** — a heavy, context-consuming sweep over hundreds of files should run in an isolated context so it doesn't pollute the main session, returning just the summary.
6. **User-level CLAUDE.md / user preferences** — a personal style preference that follows Dana across every project, not tied to any one repo.
7. **Hook** — schema version is dynamic state that goes stale in a static file; a SessionStart hook can query it live and inject the current value each session.
8. **Plain one-off prompt** — a single, non-recurring task with no reuse value doesn't warrant a skill, subagent, or persistent config.
```

One judgment call worth flagging: #7 could reasonably go in repo CLAUDE.md if the schema version changes rarely and someone's willing to keep it updated by hand — I chose the hook because it stays accurate automatically instead of relying on manual upkeep.
