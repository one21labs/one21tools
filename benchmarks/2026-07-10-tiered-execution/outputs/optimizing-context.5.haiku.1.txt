```markdown
# CLAUDE.md

## Code Style

- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase, hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Authentication

All requests to `/api/*` must carry an `X-Client-Sig` header: an HMAC-SHA256 of the request body using the key from `scripts/sign.mjs`. This is custom authentication, not Bearer auth. See `docs/auth.md` for details.

**Note:** Claude tends to write standard Bearer code here; this custom signature is required.

## Testing

- `pnpm test` launches real Chromium instances and requires the dev server running
- For iteration use `pnpm test:unit` (faster feedback loop)

## Onboarding

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open localhost:3000

## Deployment Runbook

1. Confirm main is green
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB
4. Tag release vX.Y.Z
5. `./ship staging`
6. Wait for staging healthcheck
7. Run smoke suite `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242
9. Post staging sign-off in #launch
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%
12. `./ship prod --full`
13. Verify Sentry release tagging
14. Announce in #releases
```
