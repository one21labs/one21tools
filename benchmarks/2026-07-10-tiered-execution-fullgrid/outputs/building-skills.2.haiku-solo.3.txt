# JSON Config Check

## What This Skill Does

Validates JSON configuration files for syntax errors and structural issues. When a JSON schema is available in the same directory, the skill checks the file against that schema to ensure all required fields are present and values conform to the specified types and constraints.

## When to Use

Use this skill when:
- You need to verify a JSON config file is syntactically valid before deployment
- You want to check a config file against its schema definition
- You need a report of any validation errors or missing required fields

## How It Works

The skill reads the JSON file, parses it, and reports any syntax errors. If a schema file exists in the same directory, the skill validates the configuration structure against it. Validation results are reported with line numbers and descriptions for any errors found.

## Inputs

- A JSON configuration file path
- (Optional) A JSON schema file in the same directory

## Output

A validation report showing:
- Pass/fail status
- Any syntax errors with line numbers
- Any schema violations with field names and required types
