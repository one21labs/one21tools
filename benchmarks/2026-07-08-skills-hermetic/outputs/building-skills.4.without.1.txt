```markdown
# Deploy Checks

Run before every production deployment:

1. Verify CI is green on the release commit.
2. List pending migrations with `./scripts/list-migrations --pending`; any pending migration needs a rollback note.
3. Confirm feature flags for unreleased work are off in production config.
4. Post the checklist result in the deploy thread before promoting.
```

One quick flag before you ship this: a description that broad will fire on things like "fix this typo" or "write a unit test" — tasks that have nothing to do with deploying — which teaches people to dismiss the skill on sight and likely makes the "keeps skipping the checklist" problem worse, not better. If the actual issue is that it doesn't trigger on real deploy language, a tighter fix like `description: Use before deploying, releasing, promoting, or shipping code to production.` targets the same failure mode without the false-positive noise.
