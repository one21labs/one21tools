# CLAUDE.md

## Style Guide

- **Prefer `const` over `let`** for variables that don't need reassignment.
- **Use meaningful, descriptive names** for variables, functions, and components.
- **Components in PascalCase** (e.g., `UserProfile`, `CheckoutForm`).
- **Hooks prefixed with `use`** (e.g., `useAuth`, `useFetchData`).
- **Keep functions small** — each function should do one thing well.
- **Avoid deeply nested ternaries** — extract conditions into variables or helper functions for readability.

## API Authentication

All requests to `/api/*` endpoints must carry an **`X-Client-Sig` header** containing an HMAC-SHA256 signature of the request body.

**Critical:** This is **not** Bearer authentication. Do not use standard Bearer token patterns — they will fail.

- **Header name:** `X-Client-Sig`
- **Signature algorithm:** HMAC-SHA256
- **Signing key:** Located in `scripts/sign.mjs`
- **What to sign:** The request body
- **Applies to:** All `/api/*` requests

For full implementation details and examples, see `docs/auth.md`.

## Testing

- **`pnpm test`**: Launches real Chromium instances and requires the dev server to be running (`pnpm dev`). Use for comprehensive integration testing.
- **`pnpm test:unit`**: Fast unit tests for rapid iteration during development. Does not require the dev server.

## Onboarding

1. Clone the repo
2. Run `pnpm install`
3. Copy `.env.example` to `.env`
4. Run `pnpm dev`
5. Open `localhost:3000`

## Deploy Runbook

Follow these 14 steps in order for production deployments:

1. **Confirm main is green** — verify CI/CD pipeline passes on the main branch.
2. **Build:** `pnpm build`
3. **Analyze bundle:** `pnpm run bundle-analyze` — abort the deployment if the main chunk exceeds 900kB.
4. **Tag release:** Create a git tag `vX.Y.Z` (e.g., `v1.2.3`).
5. **Deploy to staging:** `./ship staging`
6. **Wait for staging healthcheck** — monitor until staging environment reports healthy.
7. **Run smoke tests:** `pnpm smoke:staging`
8. **Manual verification:** Verify checkout flow manually using test card `4242`.
9. **Post staging sign-off:** Post message in `#launch` Slack channel confirming staging is ready.
10. **Canary deployment:** `./ship prod --canary 5` (5% of traffic)
11. **Monitor error rate:** Watch error rate for 15 minutes. If error rate exceeds 0.5%, execute rollback: `./ship rollback`
12. **Full rollout:** `./ship prod --full`
13. **Verify Sentry release:** Confirm the release is properly tagged in Sentry.
14. **Announce release:** Post announcement in `#releases` Slack channel.
