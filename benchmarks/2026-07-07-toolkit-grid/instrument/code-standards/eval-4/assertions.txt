* No exception is silently discarded: there is no bare 'except:' and no except block whose body is only pass/continue with no logging
* Each rejected feed line is logged at WARN or ERROR with the offending line content (or its sku) and the reason it was rejected, so the vendor issue is diagnosable from logs
* Malformed-line handling is per-line: a bad line is skipped and all valid lines in the same batch are still applied (the whole loop is not abandoned on first failure)
* A rejected line writes nothing into prices: no entry is created for a line whose price failed to parse or whose format was wrong (state unchanged on failure)
* Failure reporting uses the logging module with a level, not print, despite the time pressure
