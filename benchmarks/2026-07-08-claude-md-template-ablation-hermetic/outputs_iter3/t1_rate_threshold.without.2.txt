## Fastest stopgap (no app deploy needed) — if you're behind nginx

```nginx
# http block
limit_req_zone $binary_remote_addr zone=search_limit:10m rate=10r/s;

# server/location block
location /search {
    limit_req zone=search_limit burst=20 nodelay;
    limit_req_status 429;
    proxy_pass http://backend;
}
```
Reload nginx (`nginx -s reload`) — takes effect in seconds, no code deploy.

## Application-level (proper fix, per-user/per-key limiting)

**Node/Express** (using `express-rate-limit` + Redis so it works across multiple instances):
```js
const rateLimit = require('express-rate-limit');
const { RedisStore } = require('rate-limit-redis');
const redisClient = require('./redisClient'); // your existing ioredis/redis client

const searchLimiter = rateLimit({
  windowMs: 60 * 1000,
  max: 30, // 30 requests/min per key
  standardHeaders: true,
  legacyHeaders: false,
  keyGenerator: (req) => req.headers['x-api-key'] || req.ip,
  store: new RedisStore({
    sendCommand: (...args) => redisClient.call(...args),
  }),
  message: { error: 'Too many requests, slow down.' },
});

app.use('/search', searchLimiter);
```

**Python/FastAPI** (using `slowapi`):
```python

from slowapi.util import get_remote_address

@limiter.limit("30/minute")
```
