```python
import logging

logger = logging.getLogger(__name__)


def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, IndexError):
            logger.warning("Skipping malformed feed line: %r", line)
    return prices
```

Change: each line is now parsed inside a `try/except` so one bad line (wrong field count, non-numeric price, etc.) is logged and skipped instead of raising and aborting the whole batch. Nothing else touched — plan to tighten validation/error handling next sprint as you said.
