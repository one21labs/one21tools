I'll write the minimal fix that wraps the parsing in try/except to skip malformed lines:

```python
def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except ValueError:
            continue
    return prices
```

This catches:
- `ValueError` from unpacking if a line doesn't split into exactly 2 parts
- `ValueError` from `float()` if the price isn't a valid number

Malformed lines are silently skipped, and processing continues with the next line. No more batch-wide crashes.
