# Claude Code Guidelines for checkout-service

## Project Overview

**Stack:** Next.js 14 (App Router), TypeScript, Tailwind CSS, Prisma ORM on PostgreSQL  
**Ownership:** Payments squad  
**History:** Rails prototype (2021) → Next.js rewrite by platform team (2023) → current ownership

## Critical Rules

### Money Handling
- **ALL amounts must be integer cents** — never use floats
- Background: A float-rounding bug in 2024 corrupted invoices; this is a hard constraint
- When handling currency conversions or calculations, always work in cents and use integer arithmetic

### Migrations
- **ALWAYS use `./scripts/migrate.sh`** — never run `prisma migrate` directly
- Running prisma migrate bypasses the shadow database that CI relies on
- This applies to all migration operations: creating, running, and resetting

### Testing
- `npm test` — runs Playwright E2E suite (requires running dev server)
- `npm run test:unit` — runs unit tests
- E2E tests will fail without `npm run dev` active in another terminal

### Authentication
- API routes use custom `X-Session-Token` header scheme (not Bearer tokens)
- Full authentication scheme documented in `docs/api.md` — refer to it for implementation details

### Style & Code Conventions
- Reference: `docs/style-guide.md` (45 pages)
- **No default exports** — all exports must be named
- **Zod schemas colocate with their route handlers** — don't create separate schema files
- Follow all other conventions in the style guide

## Deployment Procedure

All deployments follow this 7-step process (identical across all six company repos):

1. Merge PR to main branch
2. Wait for CI to pass (all checks green)
3. Run `npm run build:prod`
4. Deploy to staging: `./deploy staging`
5. Smoke-test `/healthz` endpoint on staging
6. If p95 latency exceeds 500ms, page on-call before proceeding
7. Deploy to production: `./deploy prod`
8. Announce deployment in #releases Slack channel

## Onboarding

Full onboarding instructions are in README.md. Standard workflow:
```bash
git clone <repo>
npm install
cp .env.example .env
npm run dev
```

## Additional Context

- **CI/CD:** Relies on shadow database for testing migrations
