# CLAUDE.md

## Code Style (Python)
- 4-space indentation.
- Descriptive variable names.
- Add docstrings to all public functions.
- Keep functions short and focused; prefer early returns over deep nesting.
- Follow PEP 8.
- Prefer f-strings over `.format()`.
- Prefer composition over inheritance.
- Never use mutable default arguments.

## Error Handling & Logging
- Handle exceptions; never let them crash silently.
- Use `logging`, not `print` statements.
- Error messages must include context about what failed.

## Testing
- Write unit tests for new code.
- Run tests locally with `poetry run pytest -m "not slow"`. Tests marked `slow` require the seeded DB from `make db-seed`.

## Git & PR Workflow
- Keep pull requests small.
- Branch names use the ticket prefix, e.g. `PLAT-123-short-description`.
- Format code before committing.
- Run `make check` before every commit — CI runs the identical target.

## Security & Deployment
- Never commit secrets to the repository.
- Production deploys happen only from the `main` branch.

## Architecture Guardrails (do not "fix" these)
- All outbound HTTP goes through the wrapper in `core/http.py`; raw `requests` calls bypass the retry budget. Retries are capped at 3 attempts.
- Timestamps are stored UTC-naive on purpose — the ingestion pipeline depends on naive values. Do not convert them to timezone-aware.
- Never hand-edit generated code under `gen/`; it is regenerated via `make codegen`.
- Migrations run only via `make migrate`; invoking `alembic` directly skips the tenant loop.

## Reference
- Events table schema is at version 9.
- Config keys (defined in `settings.py`): `API_URL`, `RETRY_MAX`, `QUEUE_NAME`, `BATCH_SIZE`.

## Documentation
- Record non-obvious decisions in `docs/decisions/`.

## Needs Follow-up
- The old rule "check flags with `flags.is_enabled()` before shipping gated features" is stale: the feature-flag service was decommissioned in Q1 and `flags.py` no longer exists. Kept here so the guidance isn't lost — replace with the current gating mechanism (if any) or remove once confirmed unnecessary. Do not follow this rule as written.
