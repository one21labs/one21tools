```python
import logging

logger = logging.getLogger(__name__)


def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except ValueError:
            logger.warning("Skipping malformed feed line: %r", line)
            continue
    return prices
```

This catches the `ValueError` from a bad `split(",")` (wrong number of fields) or a non-numeric price, logs the offending line, and moves on instead of aborting the batch. Everything else is untouched — no other refactor.
