```markdown
# Project Guide for Claude

## Onboarding
1. Clone the repo
2. `pnpm install`
3. Copy `.env.example` to `.env`
4. `pnpm dev`
5. Open http://localhost:3000

## Code Style
- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase
- Hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Authentication (/api/*)
Requests to `/api/*` do NOT use Bearer tokens. They require a custom
`X-Client-Sig` header: an HMAC-SHA256 signature of the request body,
generated using the key from `scripts/sign.mjs`. Full details in
`docs/auth.md`. Do not write standard `Authorization: Bearer ...` code
for these endpoints — it is wrong and will fail.

## Testing
- `pnpm test` — full E2E suite; launches real Chromium instances and
  requires the dev server running. Slow — not for iteration.
- `pnpm test:unit` — fast unit tests. Use this while iterating.

## Deployment
Full production deploy process lives in `runbook.md`. Follow it exactly,
in order — do not skip the staging soak or canary steps.
```

**runbook.md**
```markdown
# Deploy Runbook

1. Confirm main is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release vX.Y.Z.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```
