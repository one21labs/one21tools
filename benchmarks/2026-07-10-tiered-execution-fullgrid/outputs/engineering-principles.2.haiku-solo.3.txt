# Documentation Restructuring Decisions

Here are my decisions for each piece of duplicated, contradictory, or misplaced information:

## 1. Setup Instructions (Local setup steps)
**Canonical Location:** README.md

- **What to keep:** README.md version (`clone, npm install, copy .env.example, npm run dev`) — this aligns with the "port clash" resolution mentioned in lessons learned
- **What happens to CONTRIBUTING.md copy:** Delete the duplicate setup steps (the `clone, npm ci, create .env by hand, npm start` version). Replace with a single line: "See README.md for setup instructions."

## 2. API Request Timeout Value
**Canonical Location:** src/http/client.js (source of truth)

- **What to keep in docs:** Remove the hardcoded timeout values from both README.md and CONTRIBUTING.md. Replace with a single note in README.md: "API request timeout is configured in `src/http/client.js`."
- **Why:** The actual constant lives in code; docs will contradict and drift. The 30s vs 45s contradiction proves this.
- **What happens to README.md's "30 seconds":** Delete and replace with the reference above
- **What happens to CONTRIBUTING.md's "45 seconds":** Delete entirely

## 3. Support Email
**Canonical Location:** README.md (no changes)

- This is correctly placed; it only appears once and is user-facing. Keep as-is.

## 4. billing.js recalcTotals() Rule
**Canonical Location:** CLAUDE.md

- **What to keep:** "In src/billing.js, always call recalcTotals() after any mutation to line items" — belongs in CLAUDE.md as a codebase constraint
- **Add additionally:** A one-line comment at the top of src/billing.js: `// See CLAUDE.md: always call recalcTotals() after mutations`
- **What happens to other copies:** There should be no other copy

## 5. Lessons Learned Section
**Canonical Location:** Delete from CLAUDE.md

- **Why:** CLAUDE.md should contain *current* development rules, not historical narrative. The port clash (2025-11) and export rename (2026-01) are already captured in git history.
- **What happens to it:** Delete the "Lessons learned" section from the draft CLAUDE.md. If your team wants structured decision rationale, create a separate `DECISIONS.md` (Architecture Decision Records format) — but don't put it in CLAUDE.md.

---

**Summary:** README.md becomes the single source for setup and contact info, CONTRIBUTING.md removes duplication and links to README, CLAUDE.md keeps only current code rules (no history), and src/http/client.js remains the authoritative timeout source.
