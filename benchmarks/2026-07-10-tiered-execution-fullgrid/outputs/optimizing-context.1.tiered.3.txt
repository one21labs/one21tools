# CLAUDE.md — checkout-service

## Stack

Next.js 14 (App Router), TypeScript, Tailwind, Prisma/Postgres, npm.

## Money handling

**All amounts are integer cents. Floats for money are banned.** A float-rounding incident in 2024 corrupted invoices — this is a hard constraint, not a style preference.

## Testing

- `npm test` — Playwright E2E suite. **Requires a running dev server or it fails.** (This is a non-obvious gotcha.)
- `npm run test:unit` — unit tests, no server required.

## Migrations

Always run `./scripts/migrate.sh`. Never call `prisma migrate` directly — it breaks the shadow database that CI relies on.

## Auth

API routes authenticate with a custom `X-Session-Token` header scheme (not standard Bearer tokens). Full scheme documented in [docs/api.md](./docs/api.md).

## Style

See [docs/style-guide.md](./docs/style-guide.md) (45 pages). Two explicit deviations from typical TypeScript style:
- No default exports.
- Zod schemas colocated next to their route handlers.

## Deploy procedure

This procedure is shared across all six company repos. Follow it in order:

1. Merge to main.
2. Wait for CI green.
3. Run `npm run build:prod`.
4. Run `./deploy staging`.
5. Smoke-test `/healthz`.
6. If p95 latency exceeds 500ms, page on-call.
7. Run `./deploy prod`.
8. Announce in #releases.

## Onboarding

See README.md for setup (clone, install, `.env`, `npm run dev`).
