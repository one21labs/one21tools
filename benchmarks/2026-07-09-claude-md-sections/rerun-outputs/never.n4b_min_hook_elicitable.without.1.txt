## The hook

Checks staged changes (not just the commit message) for the marker, since that's the typical use case — catching debug code you tagged so it doesn't slip into a commit:

```bash
#!/usr/bin/env bash
if git diff --cached | grep -q '^\+.*DO NOT COMMIT'; then
  echo "Commit rejected: found 'DO NOT COMMIT' marker in staged changes." >&2
  exit 1
fi
```

Save as `.git/hooks/pre-commit` and make it executable:

```bash
chmod +x .git/hooks/pre-commit
```

## Verify: clean commit succeeds

```bash
echo "normal code" > file.txt
git add file.txt
git commit -m "clean change"
```
Expected: commits normally, hook exits 0 silently.

## Verify: marker blocks the commit

```bash
echo "// DO NOT COMMIT: temp debug hack" > file.txt
git add file.txt
git commit -m "should be blocked"
```
Expected output:
```
Commit rejected: found 'DO NOT COMMIT' marker in staged changes.
```
and the commit does not happen (`git log` shows no new commit, `git status` still shows the file staged).

DO NOT COMMIT ..."`) rather than file contents, that needs a `commit-msg` hook instead — pre-commit hooks run before the message exists.
