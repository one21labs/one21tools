# layout.md

| Location | Purpose | Rationale |
|----------|---------|-----------|
| `/CLAUDE.md` | Root org-wide conventions | Org standards (conventional commits, `make check`, trunk-based dev) load on every session; kept concise with service pointers below. |
| `api/CLAUDE.md` | API service context | Warn that `api/gen/` is generated by `make proto`; engineers must never hand-edit there. Service-specific, not root-level. |
| `web/CLAUDE.md` | Web service context | Clarify `npm test` spawns real browsers (avoid in routine agent work); `npm run test:unit` for fast runs. Service-specific. |
| `pipeline/CLAUDE.md` | Pipeline service context | DAG import requires `AIRFLOW_HOME=pipeline/.airflow` precondition. Service-specific, not root-level. |
| `.claude/skills/incident-response/SKILL.md` | 20-step incident-response runbook | Interactive, invokable Skill (not static doc) so Claude can walk teams through it step-by-step on demand. SKILL.md must have YAML frontmatter: `name:` and `description:`. |
| `.mcp.json` | Analytics warehouse SQL access | Single MCP server config at repo root; access-scoped via connection credentials (read-only DB role) so only analysts effectively use it. Scope is enforced at credential/server level, not per-user in Claude Code. |

# CLAUDE.md

```markdown
# atlas Monorepo Context

## Org-Wide Standards

All services in this monorepo follow these conventions:

- **Conventional commits**: All commit messages follow the Conventional Commits spec (type: scope — description).
- **Pre-PR checks**: `make check` must pass locally before any PR is opened. Do not bypass this.
- **Trunk-based development**: No long-lived feature branches. Branch, PR, merge, delete — weekly cycle at most.

## Per-Service Guidance

Each service has its own quirks and conventions. Before working in a service, read its dedicated CLAUDE.md:

- **api/** — Go/gRPC service. See `api/CLAUDE.md` for warnings about generated code and proto workflows.
- **web/** — React/TypeScript frontend. See `web/CLAUDE.md` for test harness differences (real browsers vs. unit).
- **pipeline/** — Python/Airflow DAGs. See `pipeline/CLAUDE.md` for AIRFLOW_HOME precondition.

## Incident Response

When an incident occurs, engage the **incident-response** Skill (invoke with `/incident-response` or ask Claude to "walk me through the incident runbook"). It will guide you through the 20-step response procedure step-by-step.

## Analytics & Metrics

For product metrics questions, Claude can run live SQL queries against the analytics warehouse via the configured MCP server. Queries default to read-only access; if analysis requires broader access, escalate to the analytics team.
```
