name: json-config-check
description: Validates JSON configuration files for syntax correctness and, when a matching JSON Schema is available, checks conformance to that schema. Use when the user asks to validate, check, or lint a JSON config file, or needs help diagnosing a JSON parse or schema error.
---

# JSON Config Check

## Procedure

1. Locate the target JSON file (path given by the user, or inferred from context).

2. Parse it with Python's `json` module:
   ```python
   import json
   with open(path) as f:
       data = json.load(f)
   ```
   If parsing fails, report the `json.JSONDecodeError` message including line and column, and stop — do not attempt schema validation on invalid JSON.

3. Look for a schema, in this order:
   - A `$schema` key inside the JSON file itself (may be a URL or relative path).
   - A file named `<config-basename>.schema.json` in the same directory.
   - A file named `schema.json` in the same directory.

   If none of these exist, report that the file is syntactically valid JSON and that no schema was found — skip step 4.

4. If a schema is found, validate with the `jsonschema` package, collecting *all* errors rather than stopping at the first:
   ```python
   import jsonschema
   validator = jsonschema.Draft7Validator(schema)
   errors = sorted(validator.iter_errors(data), key=lambda e: e.path)
   ```

5. Report results:
   - **Valid, schema passed:** confirm success and note which schema file was used.
   - **Valid, no schema found:** confirm the JSON is syntactically valid; note that no schema check was performed.
   - **Valid, schema failed:** list each error with its JSON path (e.g. `properties.port`) and message.
   - **Invalid JSON:** report the parse error location and message only.

## Notes

- Requires the `jsonschema` package (`pip install jsonschema`); only needed when a schema is actually found.
- This is a read-only check — never modify the config file being validated.
```
