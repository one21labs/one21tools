```markdown
# CLAUDE.md

## Code Style
- Python: 4-space indentation, follow PEP 8.
- Use descriptive variable names.
- Add docstrings to all public functions.
- Keep functions short and focused; prefer composition over inheritance;
  prefer early returns over deep nesting.
- Prefer f-strings over `.format()`.
- Do not use mutable default arguments.
- Use `logging`, not `print` statements.
- Handle exceptions explicitly — never let them crash silently. Error
  messages should include context about what failed.

## Testing
- Write unit tests for new code.
- Run tests locally with `poetry run pytest -m "not slow"`.
  Tests marked `slow` require the seeded DB from `make db-seed`.

## Git, CI & PR Workflow
- Run `make check` before every commit; CI runs the identical target.
- Format code before committing.
- Keep pull requests small.
- Branch names use the ticket prefix, e.g. `PLAT-123-short-description`.
- Never commit secrets to the repository.
- Production deploys happen only from the `main` branch.

## HTTP & Retries
- All outbound HTTP goes through the wrapper in `core/http.py`.
  Raw `requests` calls bypass our retry budget.
- Retries are capped at 3 attempts.

## Data & Config Reference
- The events table schema is at version 9.
- Config keys: `API_URL`, `RETRY_MAX`, `QUEUE_NAME`, `BATCH_SIZE`
  (defined in `settings.py`).
- Timestamps are stored UTC-naive on purpose — do not "fix" them to
  timezone-aware; the ingestion pipeline depends on naive values.

## Generated Code
- Never edit files under `gen/` by hand; they are regenerated by
  `make codegen`.

## Migrations
- Migrations run only via `make migrate`; invoking `alembic` directly
  skips the tenant loop.

## Documentation
- Record non-obvious decisions in `docs/decisions/`.

## ⚠️ Needs team decision
- Rule (former #20): "Check flags with `flags.is_enabled()` before
  shipping gated features." The feature-flag service was decommissioned
  in Q1 and `flags.py` no longer exists — this rule is stale. Keeping it
  here until the team confirms whether to delete it or document its
  replacement.
```
