```markdown
## Style Rules
- Prefer `const` over `let`
- Use meaningful, descriptive names
- Components in PascalCase; hooks prefixed with `use`
- Keep functions small
- Avoid deeply nested ternaries

## API Auth — READ BEFORE WRITING ANY /api/* REQUEST CODE
Requests to `/api/*` do **NOT** use Bearer auth. Do not write `Authorization: Bearer ...` code for these routes — it will fail.

Instead, every request must carry an `X-Client-Sig` header: an HMAC-SHA256 signature of the request body, generated with the key from `scripts/sign.mjs`.

Full details: `docs/auth.md`.

## Tests
- `pnpm test` — launches real Chromium instances and requires the dev server running. Slow; full e2e.
- `pnpm test:unit` — use this for fast iteration while developing.

## Deploy
Deploying to production is a 14-step manual process — see `runbook.md`. Do not improvise a shorter deploy path.

Critical thresholds (so these are never missed even if runbook.md isn't opened):
- Abort the build if the main bundle chunk exceeds **900kB** (`pnpm run bundle-analyze`).
- Rollback the canary if the error rate exceeds **0.5%** during the 15-minute watch window.
```

## runbook.md

```markdown
# Deploy Runbook

1. Confirm `main` is green.
2. `pnpm build`
3. `pnpm run bundle-analyze` — abort if main chunk > 900kB.
4. Tag release `vX.Y.Z`.
5. `./ship staging`
6. Wait for staging healthcheck.
7. Run smoke suite: `pnpm smoke:staging`
8. Verify checkout flow manually with test card 4242.
9. Post staging sign-off in #launch.
10. `./ship prod --canary 5`
11. Watch error rate for 15 minutes; rollback via `./ship rollback` if > 0.5%.
12. `./ship prod --full`
13. Verify Sentry release tagging.
14. Announce in #releases.
```

Save these into `CLAUDE.md` and `runbook.md` in your project root and you're set for Friday.
