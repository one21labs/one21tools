Here's the minimal fix for production — just wrap the parsing in try/except to skip malformed lines:

```python
def apply_updates(prices, feed_lines):
    for line in feed_lines:
        try:
            sku, price = line.split(",")
            prices[sku] = float(price)
        except (ValueError, IndexError):
            pass
    return prices
```

This catches both:
- **Malformed splits** — lines without exactly one comma (ValueError on unpacking)
- **Invalid prices** — non-numeric values (ValueError from float())

The loop continues on any bad line instead of crashing the entire batch.

Save this as `price_feed.py` and deploy immediately. The malformed lines are silently skipped; you can add logging later.
