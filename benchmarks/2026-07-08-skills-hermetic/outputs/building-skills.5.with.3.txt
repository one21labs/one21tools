# Generating Release Notes

## Steps

1. Find the last release tag: `git describe --tags --abbrev=0`
2. Collect commits since that tag: `git log <tag>..HEAD --oneline`
3. Group commits by conventional-commit type prefix (one section per type).
4. Apply the type mapping and section order below.
5. Write the result as markdown to `RELEASE_NOTES.md` at the repo root.

## Commit type -> section

- `feat` -> Features
- `fix` -> Bug Fixes
- `perf` -> Performance
- `docs`, `chore`, `ci`, `build` -> omit
- `refactor` -> omit unless user-facing
- `revert` -> Reverts (also remove the reverted entry if it's in the same release)
- any type with `!` (e.g. `feat!`) -> Breaking Changes (always the first section)
- `fix(security)` or any security scope -> Security (always listed before Bug Fixes)

## Section order

Breaking Changes, Security, Features, Performance, Bug Fixes, Reverts. Omit empty sections.

## Entry format

- One line, imperative mood, no trailing period
- Link the PR number as `[#123]`

## Background

For definitions (release notes, semver, git tags) and the history of how this process evolved, see [references/background.md](references/background.md).

# Background

Release notes are documents distributed with a software release that describe the changes, improvements, and fixes included in that release. They help users and stakeholders understand what changed.

Semantic versioning (semver) is a versioning scheme in which version numbers take the form MAJOR.MINOR.PATCH, where MAJOR changes indicate breaking changes, MINOR changes indicate new backwards-compatible functionality, and PATCH changes indicate backwards-compatible bug fixes.

Git is a distributed version control system that records changes to files over time; a git tag marks a specific point in history, typically a release.

## History of This Process

Release notes used to be written by hand in Confluence, which took hours every sprint. In Q3 2025 the team moved to a shell script that dumped raw git log output, but nobody could read it. In January 2026 Priya rewrote it around conventional commits, and in March the old script was retired after the mobile team migrated. This skill replaces all of that.
